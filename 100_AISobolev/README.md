# AISobolev - блог buildinpublic

## Описание проекта
Блог о запуске IT-бизнеса с нуля в 2026. Без денег, без связей, с ИИ-инструментами.

## Концепция
- **Не про один продукт** — про весь путь в бизнесе
- **Фокус на bootstrapping** — как начать и расти без внешнего финансирования
- **ИИ как суперсила** — конкурентное преимущество соло-фаундера (не самостоятельная тема, а инструмент)
- **Buildinpublic** — открытое документирование процессов
- **Личный опыт** — реальные кейсы и решения, открытые цифры

## Темы для освещения

### Bootstrapping и старт
- Запуск IT-бизнеса без начального капитала
- Первые клиенты без бюджета на маркетинг
- Валидация идеи и MVP за минимальные деньги
- Revenue milestones: от 0 до первых $1000
- Продуктизация услуг: от консалтинга к продукту

### Соло-фаундер лайфстайл
- Жонглирование несколькими проектами одновременно
- Найм первого сотрудника / ассистента
- Работа с клиентами напрямую: как продавать себя
- Финансы: сколько стоит жить и работать на себя
- Инструменты и ИИ-стек соло-фаундера

### Честность и реальность
- Провалы, ошибки и что из них вынес
- Эмоциональные горки фаундерства
- Деньги: сколько заработал, сколько потратил (открытые цифры)
- Мотивация и выгорание

## Целевая аудитория
- Начинающие IT-предприниматели (хотят запустить, но не знают как)
- Бутстрэпперы (строят бизнес без инвестиций)
- Разработчики, мечтающие уйти на фриланс или свой продукт
- Соло-фаундеры и микро-команды
- Люди, начинающие с нуля (без денег, без связей, без "правильного" бэкграунда)

## Стандарты понятности контента

**Целевой читатель:** начинающий предприниматель, не обязательно технарь

### Чеклист перед публикацией
1. Проверить текст на англицизмы и бизнес-жаргон
2. Заменить непонятные термины на русские аналоги
3. Если термин нужен — дать объяснение в скобках

### Замены терминов
| Не использовать | Использовать |
|-----------------|--------------|
| вызывать API | выполнять действия в других программах |
| галлюцинация | выдумка, ошибка |
| операционный инцидент | сбой в работе |
| workflows | рабочие процессы |
| tools | инструменты |
| Human-in-the-Loop | человек в цепочке (с пояснением) |
| CAC | стоимость привлечения клиента |
| LTV | сколько клиент приносит за всё время |
| MRR | ежемесячная выручка от подписок |
| PMF | продукт нашёл своего клиента |
| churn | отток клиентов |
| runway | сколько месяцев продержишься без новых денег |

### Допустимые термины (с пояснением)
- **LLM** — оставляем, если объясняем (языковая модель)
- **AGI** — оставляем с расшифровкой
- **MVP** — минимальный продукт для проверки идеи
- **Bootstrap/бутстрэп** — бизнес без внешних инвестиций

### Что делает текст понятным
- Короткие предложения (до 15 слов)
- Структура с эмодзи-нумерацией
- Конкретные цифры вместо "много/мало"
- Примеры из реальной жизни
