# Гайд по стилю "AISobolev - buildinpublic"

## 1. Суть голоса

**Это не маркетинг. Это билдер, который думает вслух.**

Ваш голос — это разработчик-фаундер, который строит в реальном времени и делится тем, что работает, что не работает, и почему это важно. Не учитель, не гуру, а человек в процессе.

**Ключевые характеристики:**

- **Практик, а не теоретик** — говорите из опыта, не из книг
- **Честный, а не глянцевый** — показываете ошибки и сомнения
- **Провокационный, но не ради хайпа** — переворачиваете ожидания, чтобы заставить думать
- **Грамотный, но разговорный** — не академический текст, а живая речь

**Пример настоящего голоса:**
> "Я запустил IT-бизнес без единого рубля. И без единого знакомого в индустрии."

Сразу противоречие → интрига → читатель хочет узнать как.

## 2. Структура контента

### Формула поста: Problem → Reframe → Action

**Problem (Проблема):**

- Что не так с текущей ситуацией
- Личный опыт столкновения с этим
- Конкретика, не абстракция

**Reframe (Переосмысление):**

- Почему общепринятое мнение неполное
- Ваш взгляд, основанный на опыте
- Поворот, который меняет перспективу

**Action (Действие):**

- Что вы делаете с этим знанием
- Конкретные шаги или результаты
- Приглашение к диалогу

**Пример:**

```text
Problem: "Все говорят: найди инвестора. Без денег ничего не построишь."
Reframe: "Деньги не решают главную проблему — отсутствие клиентов."
Action: "Я запустил 3 проекта без рубля инвестиций. Первый клиент пришёл через холодное сообщение."
```

### Хуки (как начинать пост)

**1. Противоречие:**

- "Бизнес-план — самая бесполезная вещь, которую я написал." (флип ожидания)
- "Лучший маркетинг для бутстрэппера — холодные сообщения. Не реклама, не SEO, не контент." (против тренда)

**2. Интрига:**

- "Первый клиент заплатил мне $200. За то, что я делал бесплатно месяц назад." (неожиданный поворот)
- "47 сообщений. 2 ответа. 1 клиент. $500/мес." (цифры → интрига)

**3. Личная история:**

- "Вчера закрыл месяц с выручкой больше, чем моя зарплата в найме." (конкретика + эмоция)
- "Нанял ассистента. Первая неделя — хаос. Третья — свобода." (трансформация)

### Структура абзацев

- **Короткие абзацы:** 2-4 строки для Telegram, 1-2 строки для LinkedIn
- **Одна мысль = один абзац**
- **Списки для структуры:** когда больше 2 пунктов
- **Выделения:** **жирный** для главного, *курсив* для акцента

### Платформенные различия

**Telegram (RU) — основная:**

- Абзацы 2-4 строки (плотнее, чем LinkedIn)
- Поддерживает markdown полностью: **bold**, *italic*, `code`
- Можно использовать эмодзи-списки
- Более неформальный тон допустим
- Длинный формат — основная площадка для глубоких постов

**Threads (RU) — короткий формат:**

- Max 500 символов на пост
- Plain text только — без markdown (bold/italic не работают)
- 3-5 хештегов в конце поста
- Одна мысль = один пост
- Карусели (серия постов) для мульти-контента
- Подробнее: `resources/threads-guide.md`

**X (EN) — международная:**

- 280 символов на твит, threads для длинного формата
- English only — не переводить, а переписывать
- Прямой, пунктирный стиль ("builder thinking out loud" на английском)
- Quote tweets > retweets (добавляй свой take)
- Подробнее: `resources/x-guide.md`

**LinkedIn (RU/EN) — вторичная:**

- Абзацы 1-2 строки (короче, чем в Telegram)
- → стрелки для списков (LinkedIn НЕ рендерит markdown bullets)
- NO H2/H3 headers (LinkedIn НЕ рендерит markdown заголовки)
- NO markdown formatting кроме **bold** — курсив и ссылки работают плохо
- Visual separators между секциями (пустая строка достаточно)
- Алгоритм 2026 штрафует "Comment YES" CTA-механики — лучше моральная дилемма
- Лучший контент из Telegram/Threads, адаптированный под формат

### Фреймворк "Раньше/Сейчас"

Простая оппозиция для contrarian-постов. Показывает сдвиг в мышлении:

```text
Раньше: [старая модель/убеждение]
Сейчас: [новая реальность]
```

**Пример:**

```text
Раньше: хочешь бизнес → капитал, офис, команда, инвестор
Сейчас: хочешь бизнес → ноутбук, ИИ и первый клиент
```

**Почему работает:** Визуально чистый контраст. Читатель сам делает вывод. Не нужно объяснять — переход очевиден.

### Как заканчивать пост

**Иерархия окончаний (по engagement, из анализа топ-постов 2026):**

**1. Моральная дилемма / открытый вопрос** (лучший формат):

- "Если каждый инженер теперь немного продакт — что остаётся от роли продакт-менеджера?"
- "The implications for how we organize work are profound." (Mollick — 339 комментов)
- Провоцирует размышление, люди делятся своим опытом

**2. Естественный call-to-action:**

- "А вы как нашли своего первого клиента? 👇"
- "Что вас останавливает от запуска? 👇"
- Не продажа, а искренний интерес к опыту читателей

**3. Эмодзи-реакции** (для лёгких постов):

- "🤓 если уже пробовали"
- "🗿 если нет" (юмор через контраст)

**Избегать на LinkedIn в 2026:**

- ❌ "Comment YES to get..." — алгоритм штрафует механические CTA
- ❌ "Like + Repost if you agree" — низкокачественный engagement
- ❌ Опросы/polls ради охвата — LinkedIn снижает их приоритет

## 3. Характерные черты

- Максимум 1 рубленное предложение на абзац, остальные нормальной длины
- Делишься реальным процессом: скрины, цифры, что конкретно делал
- Высказываешь мнение прямо
- КАПС в хуке для акцента на ключевом слове/фразе
- Мат допустим (1-2 слова на пост, ситуативно, когда усиливает эмоцию)

## 4. Жёсткие ограничения

- **НЕ** использовать знак "=" в тексте (ни "пост = усилие", ни "AI = спам")
- Максимум 1 длинное тире (—) на весь пост
- Не больше 1 рубленного предложения подряд
- **НЕ** использовать английские слова там, где есть точный русский эквивалент: "авторы" вместо "контент-мейкеры", "ценность" вместо "value" и т.д.

## 5. Язык и тон

### Баланс: грамотность + разговорность

**Используйте:**

- "По сути" — для подведения итогов
- "Короче" — для ускорения повествования
- "Благо" — вместо "к счастью"
- "В общем" — для заключений

**Избегайте:**

- Академических конструкций
- Излишней терминологии без объяснений
- Наставнического тона ("вы должны", "необходимо понимать")

### Бизнес-термины для не-предпринимателей

**Правило:** объясняйте через применение, а не через определения.

**Плохо:**
> "LTV/CAC ratio показывает юнит-экономику retention funnel."

**Хорошо:**
> "На привлечение клиента трачу 500р. Клиент платит 3000/мес, остаётся 6 мес. Каждый клиент = 17500 чистыми."

Сразу понятно что это и зачем.

### Частотная лексика

**Бизнес-жаргон (используйте, но объясняйте):**

- "юнит-экономика", "CAC", "LTV", "MRR"
- "бутстрэп", "MVP", "PMF", "валидация"
- "пивот", "трекшн", "ранвей"

**Фразы-маркеры (используй естественно, не в каждом посте):**

- "Ну что ж"
- "Короче"
- "Погнали"
- "Другое дело, что..."
- "Я вам скажу"

**Разговорные связки:**

- "Кстати", "При этом", "Кроме того"
- "Спойлер:", "Long story short"
- "Такие дела" (философское завершение)

**Эмоциональные реакции:**

- "Ого!", "Вот блин!", "Офигеть!"
- Завуалированный мат: "хрена", "капец"

### Использование эмодзи

**Минимум: 0-1 на пост.**

**Любимые:** 😅 😁 🧙🏻‍♂️ ✊🏻

**Никогда** в начале пунктов списка.

### Юмор и провокация

**Юмор — сухой, неожиданный:**

- "3 дня в году, когда я могу позволить себе отключить голову и не думать о работе - 29, 30, 31 февраля."
- "накосячил при монтаже"
- "жаба душит" (про себя)
- "широко известно в узких кругах"

**Провокация через переосмысление:**

- "Бизнес-план — пустая трата времени" (флип ожидания)
- "Консалтинг — лучший старт и худший способ масштабировать."

**НЕ ради юмора:** если шутка не усиливает месседж — выбросьте.

## 6. Встроенный авторитет (Embedded Authority)

> Из анализа топовых LinkedIn-постов (февраль 2026)

Топовые авторы **не заявляют** экспертизу — они **демонстрируют** её через детали.

**Плохо (заявление):**

```text
"Как опытный предприниматель, я считаю..."
```

**Хорошо (встроенный авторитет):**

```text
"За 3 месяца вышел с 0 на $2000 MRR. Без инвестиций, без рекламы."
"Я веду 3 проекта параллельно: один соло, один с командой 6 человек, один на консалтинге."
"47 холодных сообщений. 2 ответа. 1 клиент на $500/мес."
```

**Как встраивать авторитет:**

- Конкретные цифры из своего опыта (не "много клиентов", а "3 клиента за 2 недели")
- Названия компаний и проектов (даёт ощущение реального мира)
- Описание процесса ("сам искал, сам продавал, сам делал")
- Финансовые данные в первых строках (revenue, расходы, прибыль)

## 7. Метрики engagement (ориентиры)

Разные форматы дают разные типы engagement:

| Формат | Лайки | Comment ratio | Шеры |
|--------|-------|---------------|------|
| **Research-backed contrarian** | Максимум (5K+) | ~6% | Средне |
| **Story + моральная дилемма** | Средне (1K) | ~26% | Низко |
| **Tutorial + CTA** | Средне (800) | ~60% | Средне |
| **Educational announcement** | Высоко (3K+) | ~2% | Максимум (6.6%) |
| **Analytical breakdown** | Средне (1.4K) | ~9% | Высоко (15%) |

**Что это значит для выбора формата:**

- Хочешь **reach и лайки** → Research + contrarian take
- Хочешь **комменты и дискуссию** → Story + моральная дилемма
- Хочешь **шеры и saves** → Tutorial или structured breakdown
- Хочешь **авторитет** → Data + Narrative + Framework (все три слоя)

## 8. Что НЕ делать (антипаттерны)

### ❌ Стилистические ошибки

**Не используйте парцелляцию ради парцелляции:**

```text
// Плохо
Причина в другом.
Совсем в другом.
Неожиданном.

```

**Не злоупотребляйте em-dashes (—):**

```text
// Плохо
Продукт — это не только код — это ещё и люди — и процессы.

```

Используйте обычные знаки препинания. Пишите просто.

**Не пишите длинными абзацами:**

- Больше 5 строк — разбейте
- Одна мысль = один абзац
- Списки для структуры

### ❌ Чего НЕ делает

- Не продаёт в лоб
- Не использует канцеляризмы и воду
- Не объясняет очевидное
- Не пишет длинные вступления
- Не ставит по 5 эмодзи на пост
- Не использует слова: является, данный, обеспечивать, трансформация, экосистема, парадигма

### ❌ Тональные ошибки

**Не учите (показывайте):**

```text
// Плохо
"Вы должны понимать, что бутстрэп — это единственный правильный путь."

// Хорошо
"Я запустил 3 проекта без рубля инвестиций. Вот что из этого вышло."

```

**Не гоните воду:**

```text
// Плохо
"В современном мире предпринимательства, где конкуренция растёт с каждым днём..."

// Хорошо
"3 месяца готовился к запуску. Клиентов — ноль."

```

**Не используйте термины без контекста:**

```text
// Плохо
"LTV/CAC ratio показывает юнит-экономику retention funnel."

// Хорошо
"На привлечение клиента трачу 500р. Клиент платит 3000/мес. Окупается за 1 неделю."
```

### ❌ Структурные ошибки

**Не начинайте с общих фраз:**

- "В наше время...", "Как известно...", "Многие эксперты..."
- Начинайте с противоречия, истории или провокации

**Не заканчивайте без диалога:**

- "Спасибо за внимание" ❌
- "А вы как решаете эту проблему? 👇" ✅

**Не пишите без структуры:**

- Каждый пост должен иметь Problem → Reframe → Action
- Без структуры — это поток сознания, а не пост

## 9. Примеры из реальных постов

> "На привлечение клиента трачу 500р — это только время на холодные сообщения.

> Клиент платит 3000/мес, остаётся в среднем 6 месяцев. Каждый клиент = 17500 чистыми."

Объяснение через конкретику, без жаргона.

>
> "Что вас останавливает от запуска? 👇"

Контраст создаёт юмор.
