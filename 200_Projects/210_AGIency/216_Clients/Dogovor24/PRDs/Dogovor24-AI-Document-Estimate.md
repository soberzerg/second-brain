# Оценка трудозатрат: AI Document Assistant для Dogovor24

> Оценка в часах разработки AI-ассистента по созданию юридических документов через чат-интерфейс.
> Стек: Go (Kratos + Ent), Neo4j / Apache AGE, PostgreSQL, Redis, React + Vite + Tailwind, Docker.

## Допущения

- Команда: 1 Go-backend разработчик, 1 AI/ML инженер (промпты, граф, оркестрация), 1 Frontend-разработчик (React), 1 DevOps (частичная занятость)
- QA — интегрировано в работу разработчиков (автотесты)
- Phase 1 — одна категория документов (напр. Договоры аренды)
- Phase 2 — добавление каждой следующей категории оценивается как отдельный инкремент
- Интеграция с AI Legal Gateway (проверка рисков) — сервис уже развёрнут по Phase 1 юр. ассистента
- Данные конструктора D24 — предоставлены в структурированном виде (API или DB dump)

## Примечания

1. **Phase 2 масштабируется линейно** — каждая новая категория ≈ 88 часов. Заказчик выбирает количество и порядок категорий.
2. **Параллелизация** — Backend, AI/ML и Frontend работают параллельно, что сокращает календарные сроки.
3. **Зависимость от AI Legal Gateway** — проверка рисков предполагает, что сервис юр. ассистента уже развёрнут. Если нет — добавить 12-16ч на mock/stub.
4. **Не включено в оценку:**
   - Юридическая экспертиза графа (верификация клауз и рисков юристами D24)
   - Подготовка данных конструктора D24 (экспорт, предоставление доступа)
   - Управление проектом (PM)
   - UX/UI дизайн (макеты Figma — предполагается, что дизайн делается параллельно или заменяется shadcn-компонентами)
5. **Буфер неопределённости** — рекомендуется +25-35% на непредвиденные задачи:
   - Сложность маппинга данных конструктора в граф (основной риск)
   - Итерации по качеству подбора клауз
   - Интеграционные проблемы с ЭДО D24
6. **LLM-инфраструктура** — затраты на API-вызовы LLM не включены, это операционные расходы.
7. **Графовая БД** — при выборе Apache AGE (PostgreSQL extension) вместо Neo4j оценка DevOps сокращается на ~8ч (нет отдельного контейнера), но возможны ограничения по производительности Cypher-запросов.
