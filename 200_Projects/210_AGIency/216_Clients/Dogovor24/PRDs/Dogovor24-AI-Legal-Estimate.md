# Оценка трудозатрат: AI Legal Gateway для Dogovor24

> Оценка в часах разработки мультиагентной AI-системы юридических консультаций.
> Стек: Go (Kratos + Ent), PostgreSQL, pgvector, Redis, Docker.

## Допущения

- Команда: 1 Go-backend разработчик, 1 AI/ML инженер (промпты, RAG, оркестрация), 1 DevOps (частичная занятость)
- QA — интегрировано в работу разработчиков (автотесты)
- UI модерации — на стороне D24, в оценку не входит
- Phase 1 — один кодекс, один специализированный агент
- Phase 2 — добавление каждого следующего кодекса оценивается как отдельный инкремент

## Примечания

1. **Phase 2 масштабируется линейно** — каждый новый кодекс ≈ 116 часов. Заказчик может выбрать количество кодексов.
2. **Параллелизация** — Backend и AI/ML работают параллельно, что сокращает календарные сроки.
3. **Не включено в оценку:**
   - UI модерации (реализация на стороне D24)
   - Юридическая экспертиза НПА (консультации с юристами D24)
   - Управление проектом (PM)
   - Получение и подготовка исторических данных из amicus
4. **Буфер неопределённости** — рекомендуется закладывать +20-30% на непредвиденные задачи (интеграционные проблемы, качество исходных данных НПА, итерации по промптам).
5. **LLM-инфраструктура** — затраты на API-вызовы LLM (GPT-4o / Claude) не включены, это операционные расходы.
