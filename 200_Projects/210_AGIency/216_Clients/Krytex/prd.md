# PRD: AI-бот для Krytex на базе AGIents

## Обзор приложения

AI-чат-бот для производителя автохимии Krytex (керамические покрытия, антидождь, абразивы, полироли), развёрнутый на платформе [[200_Projects/220_AGIents/README|AGIents]]. Бот отвечает на типовые вопросы клиентов в мессенджерах (Telegram, VK, MAX) и помогает менеджерам обрабатывать вопросы с маркетплейсов Ozon и Wildberries.

**Источник постановки:** [[Discovery Call 2026-04-16]].

## Цели и задачи

**Главная бизнес-цель:** снять нагрузку с менеджеров Krytex. Через 2–3 месяца после запуска большая часть типовых обращений должна закрываться ботом автономно; менеджеры подключаются только к сложным кейсам.

**Метрики успеха (целевые):**
- % обращений, закрытых без эскалации менеджеру
- Сокращение среднего времени ответа клиенту (особенно на маркетплейсах)
- Количество автоматически обработанных Q&A в день

**Анти-цель:** НЕ увеличивать нагрузку на менеджеров за счёт разгребания претензий после галлюцинаций бота. Лучше честно эскалировать, чем выдумать ответ.

## Целевая аудитория

**Заказчик (B2B-клиент AGIents):** Krytex — производитель автохимии, продаёт через маркетплейсы и свои каналы.

**Конечные пользователи бота:**

**Основной сегмент — покупатели автохимии:**
- Автовладельцы, детейлеры, мастера автомоек
- Задают вопросы в Telegram/VK/MAX и на маркетплейсах
- Pain points: долгий ответ продавца на маркетплейсе, неясность применения продукта, выбор между похожими средствами

**Вторичный сегмент — менеджеры Krytex:**
- Работают с входящими из всех каналов
- Pain points: десятки однотипных вопросов в день на маркетплейсах, повторение одних и тех же объяснений, ручная публикация ответов

**User Persona (клиент):** Детейлер или автовладелец, пишет в карточке товара на Ozon "подойдёт ли VORSIC SIC для матового ЛКП?" — ожидает ответ в течение часа, сравнивает с конкурентами. Или в Telegram Krytex: "дворники прыгают после MegaGlass, что делать?"

**User Persona (менеджер Krytex):** 2–3 человека, обрабатывают входящие из всех каналов, хотят тратить время только на сложные кейсы и продажи, а не на "в комплекте идёт губка?"

## Основные фичи

### MVP (Фаза 1–3, 6 недель)

#### Клиентский бот в мессенджерах
1. **Telegram, VK, MAX** — бот отвечает клиентам в трёх мессенджерах по единой базе знаний Q&A.
2. **Сессионная память** — бот помнит последние N сообщений текущей сессии (уже в платформе), поддерживает многошаговый диалог.
3. **Автономные ответы с защитой от галлюцинаций** — LLM с жёсткой инструкцией "не нашёл в базе — не придумывай, скажи клиенту 'передам менеджеру' и эскалируй". Настраиваемый порог / системный промпт — через существующую админку AGIents.
4. **Эскалация менеджеру** — если бот не уверен или клиент явно просит человека, случай попадает в лог escalations.

#### Редактор базы знаний
5. **Веб-редактор Q&A в админке AGIents** — CRUD по таблице Q&A, поиск, теги/категории, история изменений с возможностью отката. Менеджеры Krytex ведут базу сами.

#### Escalations
6. **Лог escalations** — отдельная вьюха в веб-админке: только диалоги, где бот не справился. Контекст: вопрос клиента + предложенный ботом ответ + причина эскалации.
7. **Нотификации в менеджерский Telegram-бот** — когда возникает новый escalation, в Telegram-бот менеджеров падает сообщение с превью и ссылкой на веб-админку. **Отвечать можно только из веб-админки** (единый source of truth, удобный аудит).

#### Маркетплейс-pipeline (Ozon + WB)
8. **Импорт новых Q&A с маркетплейсов** — бот читает новые (ещё не отвеченные) вопросы с Ozon и Wildberries через API seller-кабинетов.
9. **Менеджерский Telegram-бот** — новые Q&A падают в Telegram менеджерам Krytex с черновиком ответа от LLM. Менеджер использует черновик и **сам публикует ответ** вручную в кабинете маркетплейса (бот не пишет в Ozon/WB).
10. **Обогащение базы знаний — два пути:**
    - **Вручную через админку AGIents** — менеджер добавляет/правит Q&A в веб-редакторе.
    - **Автоматический импорт уже отвеченных Q&A с Ozon/WB** — бот по расписанию читает через API те вопросы, на которые менеджеры уже ответили в кабинетах маркетплейсов, и подгружает их в таблицу базы знаний. Каждый ручной ответ на маркетплейсе автоматически становится будущим автоответом бота.

#### Платформенные возможности (уже реализованы в AGIents)
Перечислены для полноты скоупа — отдельной разработки не требуют:
- Inbox всех диалогов с фильтрами и поиском
- Аналитика (вопросы, % автономных ответов, токены)
- Настройки бота (системный промпт, режим auto/HITL, расписание)
- Управление пользователями и ролями (админ / менеджер / читатель)
- Сессионная память

Подробнее: [[Platform Capabilities]].

### Фаза 2 (после валидации MVP)

1. **Расширение базы знаний** — инструкции к продуктам + каталог товаров (состав, характеристики) как вторая и третья базы. Закрывает кейсы типа "твёрдость VORSIC SIC" и "инструкция по керамическому молочку", провалившиеся на demo.
2. **Отзывы как источник знаний** — парсинг отзывов с Ozon/WB, LLM-нормализация, предложение добавить в базу (без автоматического ответа на сами отзывы).
3. **Мультиагентная архитектура** — когда баз знаний становится 3+, переход к специализированным агентам (consultant, instructor, catalog-search) с маршрутизацией.
4. **Дополнительные каналы** — виджет на сайте Krytex, Instagram Direct, WhatsApp (через Pact).

### Будущее развитие

- Интеграция с CRM Krytex (идентификация клиента по заказу, персонализация)
- Проактивные сценарии: follow-up после покупки, инструкции по применению
- Голосовые ответы в мессенджерах
- Перенос best practices Krytex на другие категории B2B-клиентов AGIents

## Технический стек

Стек определяется архитектурой платформы AGIents; под Krytex используем существующие решения.

| Компонент | Используем | Обоснование |
|-----------|------------|-------------|
| Платформа | AGIents | Собственная no-code B2B-платформа ИИ-агентов |
| Клиентские каналы | Telegram, VK, MAX | По запросу Krytex; Telegram уже готов, VK/MAX — интеграция |
| Менеджерский канал | Telegram (отдельный бот) | Менеджеры уже живут в Telegram |
| Маркетплейс API | Ozon Seller API, WB Seller API | Read-only: только чтение Q&A, без публикации |
| LLM | Провайдер платформы AGIents | Наценка на токены — часть биллинг-модели |
| Веб-админка | AGIents (существующая) | Редактор Q&A и лог escalations — новые секции в той же админке |
| Аутентификация | AGIents (существующая) | Роли админ / менеджер / читатель |

## Этапы разработки

### Этап 1 (Фаза 1): Telegram + Q&A + Escalations (2 недели)
- [ ] Tenant "Krytex" в AGIents, базовая настройка
- [ ] Подключение Telegram-канала (клиентский бот)
- [ ] Веб-редактор Q&A: CRUD, поиск, теги, история изменений
- [ ] Начальное наполнение Q&A из существующей таблицы Krytex (миграция)
- [ ] Лог escalations: вьюха в веб-админке, ответ менеджера → отправка клиенту в Telegram
- [ ] Менеджерский Telegram-бот: нотификации об escalations со ссылкой на веб-админку
- [ ] Настройка системного промпта: "не знаю — эскалируй"
- [ ] Pilot-демо Krytex, сбор feedback

### Этап 2 (Фаза 2): VK + MAX (2 недели)
- [ ] Интеграция VK (сообщество Krytex)
- [ ] Интеграция MAX
- [ ] Unified inbox и логика escalations работают для всех трёх каналов
- [ ] Тестирование с несколькими реальными клиентами

### Этап 3 (Фаза 3): Ozon + WB + маркетплейс-pipeline (2 недели)
- [ ] Интеграция Ozon Seller API (чтение Q&A)
- [ ] Интеграция WB Seller API (чтение Q&A)
- [ ] Периодический опрос (polling) новых Q&A
- [ ] Менеджерский Telegram-бот: доставка Q&A с черновиками ответа
- [ ] Автоимпорт уже отвеченных Q&A с Ozon/WB в базу знаний (по расписанию)
- [ ] Производственный запуск, договор на подписку

### Этап 4 (после MVP): расширение
- [ ] Подключение инструкций и каталога как дополнительных источников
- [ ] Парсинг отзывов
- [ ] Мультиагентная архитектура при 3+ базах
- [ ] Новые каналы (виджет, Instagram, WhatsApp)

## Потенциальные вызовы и решения

| Вызов | Влияние | Предложенное решение |
|-------|---------|---------------------|
| Галлюцинации LLM (демо показало 2+ неточных ответа) | Высокое — риск претензий клиентов | Жёсткий промптинг "не знаю — эскалируй", настраиваемый confidence-порог, аудит escalations в первые недели |
| Ozon Premium Plus / WB API-доступ может отсутствовать у Krytex | Высокое для фазы 3 | Зависимость клиента зафиксирована в договоре; фаза 3 не стартует без доступов; в PRD обозначено как внешний риск |
| Низкое качество стартовой Q&A-базы | Среднее — бот будет часто эскалировать, эффект "снятия нагрузки" замедлится | Фаза 1 включает миграцию + ревью существующей таблицы; менеджеры дополняют базу из маркетплейс-pipeline с первой недели |
| Стоимость токенов при росте нагрузки | Среднее — клиент может испугаться счёта | Сессионная память с таймаутом (дешевле full-history); biller shows breakdown; alert при превышении порога |
| Два интерфейса для менеджера (Telegram + веб) — путаница | Низкое | Чёткое разделение: Telegram = пуш + ссылка, веб = ответ. Документация для менеджеров Krytex на старте |
| VK/MAX — менее стабильные API | Среднее | Фаза 2 включает buffer на отладку; канал-specific абстракция в платформе AGIents |
| Менеджеры Krytex саботируют/игнорируют escalations | Среднее — escalation SLA ломается | SLA-таймеры в логе escalations, эскалация к админу Krytex, аналитика времени ответа |

## Модель биллинга

**Структура:** Fixed setup + usage-based subscription.

- **Setup (внедрение):** фиксированная цена, привязанная к скоупу фаз 1–3. Оплачивается по этапам (приёмка фазы).
- **Subscription (владение):** ежемесячная оплата по факту расхода токенов LLM с наценкой. Включает: хостинг бота в AGIents, поддержку каналов, техподдержку по SLA.

**Ориентир (из discovery):** ~500₽ за 100 вопросов, ~5000₽/мес за 1000 запросов. Конкретные цифры — в [[Коммерческое предложение]] для Krytex.

## Внешние зависимости

- **Доступы к маркетплейсам:** Ozon Premium Plus (если требуется), WB Seller API — обеспечивает Krytex.
- **Наполнение Q&A-базы:** базовое наполнение (миграция существующей таблицы) + регулярное пополнение — ответственность Krytex. AGIency помогает с миграцией и даёт инструмент, но не отвечает за содержание.
- **Одобрение промпта и тональности:** Krytex утверждает системный промпт и стиль ответов перед запуском.

## Будущие возможности

- **Расширение на другие B2B-клиенты AGIency.** Паттерн "mesенджеры + маркетплейсы + менеджерский pipeline" переиспользуется для производителей потребительских товаров. См. [[Кейс Amaks]], [[mediashop24]].
- **Self-service onboarding в AGIents.** Если пайплайн Krytex стабилизируется, можно вынести как template для новых клиентов AGIents без ручного внедрения.
- **Монетизация через маркетплейс-автоматизацию.** Ozon/WB pipeline — отдельная ценность, которую можно продавать независимо от чат-бота.
- **Генерация контента для маркетплейсов.** Из базы знаний + отзывов — автогенерация описаний товаров, FAQ-блоков карточек.
