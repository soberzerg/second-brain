## ЭТАП 1: База знаний

### 1.1 Парсинг сайта (автоматический)

**Источник:** https://miaclinic.ru
**Инструмент:** Встроенный парсер AGIents.pro

#### Что парсим
- [ ] Главная страница — общая информация о клинике
- [ ] Услуги — список всех стоматологических услуг
- [ ] Цены — прайс-лист
- [ ] Врачи — информация о специалистах
- [ ] Контакты — адрес, телефон, часы работы
- [ ] Акции — текущие спецпредложения
- [ ] О клинике — история, лицензии, оборудование

#### Ожидаемый результат
- Структурированная база знаний по услугам
- Актуальные цены (требует периодического обновления)
- Информация для консультаций

### 1.2 Документы от клиента (ручная загрузка)

**Запросить у клиента:**

| Документ | Приоритет | Описание |
|----------|-----------|----------|
| Прайс-лист (актуальный) | 🔴 Высокий | PDF/Excel с ценами |
| Памятки пациентам | 🟡 Средний | Подготовка к процедурам |
| FAQ внутренний | 🟡 Средний | Частые вопросы от пациентов |
| Противопоказания | 🟡 Средний | По каждой услуге |
| Скрипты операторов | 🟢 Низкий | Как отвечают на звонки сейчас |

### 1.3 Q&A из записей звонков (если нужно)

**Когда применяем:** Если после этапов 1.1 и 1.2 качество ответов недостаточное

#### Процесс
1. Запросить записи звонков у клиента (последние 50-100)
2. Транскрибировать (Whisper API)
3. Извлечь пары вопрос-ответ
4. Добавить в базу знаний как FAQ

#### Что это даёт
- Реальные формулировки вопросов пациентов
- Проверенные ответы от операторов
- Покрытие edge cases

## ЭТАП 2: Архитектура агентов

### Схема взаимодействия

```
                        ┌─────────────────┐
                        │   VK Сообщество │
                        └────────┬────────┘
                                 │
                                 ▼
                    ┌────────────────────────┐
                    │     Router Agent       │
                    │  (определение интента) │
                    └────────────┬───────────┘
                                 │
              ┌──────────────────┼──────────────────┐
              ▼                  ▼                  ▼
     ┌────────────────┐ ┌────────────────┐ ┌────────────────┐
     │  Консультант   │ │ Запись на приём│ │   Эскалация    │
     │    Agent       │ │    Agent       │ │  (оператору)   │
     └───────┬────────┘ └───────┬────────┘ └────────────────┘
             │                  │
             ▼                  ▼
     ┌────────────────┐ ┌────────────────┐
     │  База знаний   │ │   МИС API      │
     │    (RAG)       │ │  (расписание)  │
     └────────────────┘ └────────────────┘
```

### 2.1 Router Agent

**Задача:** Определить намерение пользователя

| Интент | Триггеры | Действие |
|--------|----------|----------|
| `consultation` | "сколько стоит", "что такое", "как подготовиться" | → Консультант |
| `booking` | "записаться", "свободное время", "к врачу" | → Запись на приём |
| `escalation` | жалоба, сложный вопрос, "позвать человека" | → Оператор |

### 2.2 Агент-консультант

**Задачи:**
- Ответы на вопросы об услугах
- Консультация по ценам
- Информация о врачах
- Подготовка к процедурам
- FAQ

**Источник данных:** RAG по базе знаний

**Примеры диалогов:**

```
Пользователь: Сколько стоит отбеливание?
Бот: Отбеливание зубов в MiaClinic стоит от 15 000 ₽.
     Мы используем систему ZOOM 4. Процедура занимает около 1 часа.
     Хотите записаться на консультацию?
```

### 2.3 Агент записи на приём

**Задачи:**
- Уточнить услугу
- Узнать предпочтения по врачу (если есть)
- Показать доступные слоты
- Создать запись в МИС
- Подтвердить запись

**Интеграция:** МИС клиника (API)

**Флоу записи:**

```
1. Уточнение услуги
   Бот: На какую услугу хотите записаться?
   User: Чистка зубов

2. Выбор врача (опционально)
   Бот: Есть ли предпочтения по врачу?
   User: Нет, любой

3. Выбор времени
   Бот: Доступные слоты на ближайшую неделю:
        • Пн 27.01 — 10:00, 14:00, 16:30
        • Вт 28.01 — 11:00, 15:00
        Какое время удобно?
   User: Понедельник 14:00

4. Сбор данных
   Бот: Ваше имя?
   User: Иван Петров
   Бот: Номер телефона для подтверждения?
   User: +7 999 123 45 67

5. Подтверждение
   Бот: Отлично! Вы записаны:
        📅 Пн, 27 января в 14:00
        🦷 Профессиональная чистка
        📍 MiaClinic, ул. Примерная, 1

        За день до приёма мы напомним вам SMS.
        До встречи!
```

## ЭТАП 3: Интеграции

### 3.1 VK Сообщество

**Тип интеграции:** VK Bot API

#### Настройка
- [ ] Получить access_token сообщества
- [ ] Настроить Callback API
- [ ] Включить возможности бота в настройках сообщества

#### Функции VK бота
- Приём сообщений
- Отправка текста
- Клавиатуры (inline buttons)
- Карточки услуг (карусели)

#### Пример клавиатуры

```
┌─────────────────────────────────────┐
│         Чем могу помочь?            │
├─────────────────────────────────────┤
│  [Услуги и цены]  [Записаться]     │
│  [Наши врачи]     [Как добраться]  │
└─────────────────────────────────────┘
```

### 3.2 МИС (Медицинская информационная система)

**Задача:** Интеграция для записи на приём

#### Что нужно узнать у клиента
- [ ] Какая МИС используется? (МЕДИАЛОГ, ИНФОКЛИНИКА, 1С:Медицина, своя?)
- [ ] Есть ли API?
- [ ] Возможность создания записей через API
- [ ] Получение расписания врачей
- [ ] Документация

#### Необходимые API методы

| Метод | Описание |
|-------|----------|
| `GET /doctors` | Список врачей |
| `GET /services` | Список услуг |
| `GET /schedule/{doctor_id}` | Расписание врача |
| `GET /slots/{date}` | Доступные слоты |
| `POST /appointments` | Создание записи |
| `GET /appointments/{id}` | Статус записи |

#### Fallback (если нет API)
- Бот собирает заявку
- Отправляет оператору в отдельный чат
- Оператор создаёт запись вручную
