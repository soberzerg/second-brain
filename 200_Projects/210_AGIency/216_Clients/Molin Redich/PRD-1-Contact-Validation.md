# PRD-1: Валидация базы контактов

**Клиент:** Molin Redich
**Дата:** 2026-03-07
**Статус:** Черновик для оценки

---

## Проблема

У клиента есть табличная база компаний. Текущий процесс обогащения данных (поиск сайта по ИНН/названию, парсинг контактов, сверка) реализован через скрипты, работает медленно и нестабильно. Нужно оптимизировать скорость и качество обработки.

## Решение

Сервис автоматического обогащения контактной базы:

1. **Поиск официального сайта** по ИНН и/или названию компании
2. **Парсинг контактов** с найденного сайта (email, телефон)
3. **Сопоставление** найденных данных с существующими в базе
4. **Отчёт** с результатами: новые контакты, расхождения, не найденные

## Функциональные требования

### Входные данные
- Табличный файл (Excel/CSV) с полями: ИНН, название компании, существующие контакты

### Поиск сайта
- Поиск по ИНН через открытые реестры (egrul.nalog.ru, rusprofile.ru и аналоги)
- Поиск по названию компании через поисковые API (Google/Yandex)
- Ранжирование результатов для определения наиболее вероятного официального сайта
- Обработка случаев: несколько сайтов, сайт не найден, редирект

### Парсинг контактов
- Извлечение email-адресов со страниц сайта (главная, контакты, о компании)
- Извлечение телефонов (с нормализацией форматов)
- Глубина обхода: до 3-5 страниц на сайт (настраиваемо)
- Обход типовых защит (rate limiting, капча — в разумных пределах)

### Сопоставление
- Сравнение найденных контактов с данными из базы клиента
- Отметки: совпадение / новый контакт / расхождение
- Выходной файл с обогащёнными данными

### Скорость и масштаб
- Параллельная обработка (пул воркеров)
- Целевая скорость: 500-1000 компаний/час (зависит от внешних лимитов)
- Возобновление после сбоя (checkpoint)

## Допущения (уточнить с клиентом)

- Формат и структура текущей базы данных
- Объём базы (тысячи? десятки тысяч записей?)
- Нужен ли веб-интерфейс или достаточно CLI/скрипта
- Регулярность запусков (разово? еженедельно?)
- Бюджет на внешние API (поисковые, реестры)

## Этапы реализации

| Этап | Описание | Срок |
|------|----------|------|
| 1 | Анализ текущих скриптов и данных клиента, проектирование | 1 |
| 2 | Модуль поиска сайта (ИНН + название) | 2 |
| 3 | Модуль парсинга контактов с сайтов | 2 |
| 4 | Модуль сопоставления и формирование отчёта | 4 |
| 5 | Оптимизация скорости | 4 |
| 6 | Тестирование на реальных данных | 3 |

**Общий срок: 1-2 недели (16 часов)**

## Оценка бюджета

**80 000 - 150 000 руб.**

Диапазон зависит от:
- Объёма базы и сложности данных
- Необходимости веб-интерфейса (+ к стоимости)
- Интеграции с существующими системами клиента
- Необходимости платных API (egrul, поисковые)

## Что НЕ входит

- Регулярное обслуживание и обновление парсеров
- CRM-интеграция (отдельная задача)
- Массовая рассылка по найденным контактам
- Обход капчи и серьёзных защит от парсинга
