# PRD-3: AI-помощник для сейлзов

**Клиент:** Molin Redich
**Дата:** 2026-03-07
**Статус:** Черновик для оценки

---

## Проблема

Менеджеры по продажам тратят время на поиск информации об услугах компании, ценах, условиях. Нет единой точки доступа к актуальной базе знаний. Клиент также хочет "онлайн-помощь" — подсказки менеджеру прямо во время коммуникации с клиентом.

## Решение

AI-ассистент для отдела продаж с двумя режимами:

1. **Чат-бот с базой знаний** — менеджер задаёт вопрос, получает ответ на основе внутренних документов (RAG)
2. **Онлайн-помощник** — подсказки и рекомендации в реальном времени при общении с клиентом

## Функциональные требования

### Чат-бот с базой знаний (RAG)

#### База знаний
- Загрузка документов: PDF, Word, Excel, текстовые файлы
- Автоматическое индексирование и векторизация
- Обновление базы: добавление/удаление документов через интерфейс
- Типы контента: прайс-листы, описания услуг, FAQ, скрипты, кейсы, условия договоров

#### Чат-интерфейс
- Веб-интерфейс (доступен с компьютера и телефона)
- Ответы с указанием источника (ссылка на документ/раздел)
- История диалогов
- Возможность уточняющих вопросов в контексте

#### Примеры запросов
- "Какие условия по услуге X для компаний с оборотом от 100 млн?"
- "Чем наше предложение отличается от конкурента Y?"
- "Какие кейсы есть в отрасли клиента?"
- "Подготовь краткое описание услуги Z для письма клиенту"

### Онлайн-помощник (расширение)

#### Режим подсказок
- Интеграция с каналом коммуникации (чат/мессенджер)
- Анализ входящего сообщения клиента в реальном времени
- Генерация рекомендуемого ответа / подсказки для менеджера
- Менеджер решает: использовать подсказку, отредактировать или проигнорировать

#### Сценарии
- Клиент задаёт вопрос о продукте — бот подсказывает ответ из базы знаний
- Клиент выдвигает возражение — бот предлагает контраргумент
- Клиент упоминает конкурента — бот показывает сравнение

## Допущения (уточнить с клиентом)

- Какие документы составляют базу знаний (объём, форматы)?
- Где менеджеры общаются с клиентами (Telegram, WhatsApp, CRM-чат, email)?
- Нужен ли Telegram-бот или веб-интерфейс (или оба)?
- Сколько менеджеров будут использовать?
- Какой приоритет: чат-бот с базой (быстрее) или онлайн-помощник (сложнее)?
- Где размещать (сервер клиента)?
- Есть ли требования к конфиденциальности данных?

## Этапы реализации

| Этап | Описание | Срок |
|------|----------|------|
| 1 | Сбор и подготовка базы знаний, проектирование | 3-5 дней |
| 2 | RAG-пайплайн: индексация, поиск, генерация ответов | 1-1.5 недели |
| 3 | Чат-интерфейс (веб и/или Telegram) | 1 неделя |
| 4 | Тестирование на реальных вопросах, тюнинг качества | 3-5 дней |
| 5 | Онлайн-помощник: интеграция с каналом коммуникации | 1-1.5 недели |

**Общий срок: 4-6 недель**

Если приоритет — только чат-бот (без онлайн-помощника): **3-4 недели**

## Оценка бюджета

**200 000 - 350 000 руб.**

| Вариант | Бюджет | Срок |
|---------|--------|------|
| Только чат-бот с базой знаний | 120 000 - 180 000 руб | 3-4 недели |
| Чат-бот + онлайн-помощник | 200 000 - 350 000 руб | 4-6 недель |

Диапазон зависит от:
- Объёма и сложности базы знаний
- Количества каналов для онлайн-помощника
- Требований к интерфейсу
- Глубины интеграции с существующими системами

### Операционные расходы (ежемесячно)
- LLM API (генерация ответов): 3-10 тыс. руб/мес (зависит от числа запросов)
- Embedding API (индексация): 1-3 тыс. руб/мес
- Серверная инфраструктура: зависит от размещения

## Что НЕ входит

- Создание контента для базы знаний (документы предоставляет клиент)
- Автоматическая отправка ответов клиентам (только подсказки менеджеру)
- Голосовой ассистент (только текст)
- CRM-интеграция (отдельная задача, если потребуется)
- Обучение менеджеров работе с продуктом (только инструкция и демо)
