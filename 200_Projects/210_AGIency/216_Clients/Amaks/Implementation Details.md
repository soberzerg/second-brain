## Функциональность

### Реализованные фичи

#### 1. Консультирование

**Описание:**
- Ответы на вопросы об услугах
- Информация о лечебных программах
- Условия проживания
- Ценообразование

**Технология:**
- RAG на базе документации санатория
- Semantic search по FAQ
- Контекстные ответы с ссылками на источники

#### 2. Проверка доступности

**Описание:**
- Проверка свободных номеров
- Информация о датах
- Категории номеров

**Интеграция:**
- API PMS системы
- Real-time проверка
- Кеширование: _есть ли?_

#### 3. Бронирование

**Реализация:**
- Полное бронирование: _да/нет?_
- Пре-бронирование (резерв): _да/нет?_
- Передача оператору: _да/нет?_

**Процесс:**
1. Сбор требований клиента
2. Проверка доступности
3. Создание брони (или запрос оператору)
4. Подтверждение (email/SMS)

#### 4. Эскалация

**Триггеры эскалации к оператору:**
- Сложный запрос
- Недовольство клиента
- Специфические требования

## Диалоговые сценарии

### Основные флоу

#### 1. Общая консультация

```
User: Расскажите о вашем санатории
Bot: [Общая информация]
Bot: Чем могу помочь? (услуги/цены/бронирование)
User: [Выбор]
Bot: [Соответствующая информация]
```

#### 2. Бронирование

```
User: Хочу забронировать номер
Bot: [Сбор дат]
Bot: [Сбор количества гостей]
Bot: [Сбор категории номера]
Bot: [Проверка доступности]
Bot: [Предложение вариантов]
User: [Выбор]
Bot: [Подтверждение + контактные данные]
Bot: [Создание брони]
Bot: [Подтверждение по email/SMS]
```

#### 3. FAQ

```
User: Как добраться?
Bot: [Ответ из базы знаний с источниками]
Bot: Могу ли еще чем-то помочь?
```
