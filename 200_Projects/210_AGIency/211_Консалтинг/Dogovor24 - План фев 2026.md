## Платформа Dogovor24 — План на 3 месяц

---

## Фаза 1: Стабилизация

### Цель

Устранить критические уязвимости безопасности, предотвратить потенциальные инциденты. Дополнительно: устранить критические проблемы производительности.

### Работы (предварительная оценка объема)

| Наименование работ, результат | Объем, часов |
| :---- | :---- |
| Презентация, обучение команды разработке с использованием AI \+ руководство | 6 |
| Аудит производительности по основным сервисам (\~3-6 сервисов) | 16 |
| Рекомендации по внедрению трейсинга | 4 |
| Создание тикетов PHP-разработчикам (безопасность, производительность) | 6 |
| Создание тикетов DevOps-специалистам (логирование, мониторинг, трейсинг, алерты) | 4 |
| Консультирование команды по реализации рекомендаций | 4 |

### Ключевые активности

Неделя 1

- Рекомендации по внедрению трейсинга  
- Аудит производительности и отчёт по основным сервисам (по приоритету)  
- Создание тикетов  
- Консультирование команды по реализации исправлений

Неделя 2

- Создание руководства по разработке с использованием AI  
- Консультирование команды по реализации исправлений

Неделя 3

- Презентация, обучение команды разработке с использованием AI  
- Аудит, создание тикетов, консультирование команды

Неделя 4

- Аудит, создание тикетов, консультирование команды

### Критерии завершения фазы

- [ ] Завершен аудит производительности для основных 4+ сервисов  
- [ ] Созданы тикеты команде для реализации базового логирования security-событий  
- [ ] Созданы тикеты команде для реализации исправлений по безопасности и производительности  
- [ ] Созданы тикеты DevOps для настройки необходимого ПО  
- [ ] Созданы тикеты DevOps для настройки алертов на подозрительную активность  
- [ ] Проведено обучение команды базовым практикам разработки с использованием AI
