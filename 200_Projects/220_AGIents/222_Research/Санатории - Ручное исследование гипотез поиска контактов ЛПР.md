# Санатории — ручное исследование гипотез поиска контактов ЛПР

**Контекст:** AGIents готовится к outbound-кампаниям по санаториям. Перед закладкой автоматизации и подключением платных enrichment-источников нужно вручную проверить, какие открытые источники реально дают качественные контакты ЛПР, а какие — пустышка. Цель — дать практичную базу для решения «что автоматизировать в первую очередь».

**Что считать ЛПР:** генеральный директор (CEO), директор по развитию / коммерческий директор, директор по маркетингу / руководитель отдела продаж, иногда главный врач (для медицинских вопросов) или замдиректора по медчасти.

**Терминология:**
- **Direct-контакт** — персональный канал связи с конкретным ЛПР (личный email, личный номер, личный профиль).
- **Fallback-контакт** — общий канал санатория (info@, телефон ресепшена, общая страница в соцсети) или контакт не-ЛПР (HR, секретарь), через которого можно эскалировать.

## 2. Протестированные гипотезы и yield

Каждая гипотеза проверена на всех 5 санаториях, если не указано иное. Yield = количество санаториев из 5, где гипотеза дала хоть один полезный сигнал.

### H1. Официальный сайт — страницы «Контакты», «Руководство», «О санатории»

**Шаги:** Google-поиск офсайта → переход на /kontakty/ или /contacts/, /о-санатории/, /руководство/, /документы/. Сбор всех email, телефонов, ФИО.

**Просмотрено записей:** 5 сайтов × 2-3 страницы каждый = ~12 страниц.

**Yield:** 5/5 — на каждом сайте нашлось что-то.
- ФИО руководителя/ЛПР с офсайта: 2/5 (Янган-Тау — Акбашев А.Р.; Машук — конфликтующая запись с О.А. Боковым на главной vs Карлова М.Г. в ЕГРЮЛ).
- Персональный email с офсайта: 1/5 (Знание — `marketing@skk-znanie.ru` приписан к гендиру косвенно через раздел «Документы и реквизиты»).
- ФИО мед-руководителей и врачей: 1/5 (Загорские дали — 18 ФИО+должность).
- Email отделов (sales@, secr@, b2b@, marketing@, market@): 5/5 — у всех нашёлся хотя бы один.
- Телефоны отделов: 5/5.
- Ссылки на соцсети: 4/5 (Полтава-Крым — нет в подвале).

**Типы получаемых контактов:** общий email, email отдела (включая marketing/sales/b2b — близко к ЛПР), телефон отдела, имена мед-персонала, ссылки на соцсети, юр. реквизиты.

**Ограничения:**
- ФИО гендира редко указано на офсайте (это не их формат коммуникации).
- Загорские дали как государственное учреждение охотно публикует мед-персонал, но прячет руководство по умолчанию.
- На частных сайтах (Машук, Знание) встречаются устаревшие имена и противоречия с ЕГРЮЛ — нельзя доверять без верификации.
- Полтава-Крым — единственный сайт без соцсетей в подвале и без ФИО руководства.

**Где не сработало:** офсайт не даёт ФИО гендира на 4 из 5; для Полтава-Крым — не дал вообще ничего по руководству.

### H2. ЕГРЮЛ / RusProfile / list-org / checko / saby (открытые части)

**Шаги:** Google-поиск «<санаторий> rusprofile ИНН директор» → открыть карточку компании → забрать ФИО руководителя, должность, дату назначения, ИНН, ОГРН, юр. адрес, учредителей.

**Просмотрено записей:** 5 организаций (плюс по 2-3 карточки на разных агрегаторах для перекрёстной верификации).

**Yield:** 5/5 на ИНН и юр. реквизитах. 5/5 на ФИО руководителя (с разными уровнями актуальности).

**Типы получаемых контактов:** ФИО+должность руководителя, ИНН/ОГРН (для дальнейшего поиска в zakupki, hh, банках), дата назначения (важна для сегментации — новые директора активнее меняют процессы), юр. адрес, учредители (даёт картину «частный/государственный/корпоративный»).

**Ограничения:**
- Email и телефон в открытой части обычно скрыты («требуется профессиональный доступ»).
- ФИО руководителя — это формальный руководитель в ЕГРЮЛ, не всегда совпадает с реально активным лицом (для коммерческих переговоров чаще нужны ком. директор/директор по развитию, которых в ЕГРЮЛ нет).
- В случае Полтава-Крым формальный руководитель — это руководитель временной администрации (компания в санации/банкротстве). Это важный сигнал: возможно, для переговоров надо искать конкурсного управляющего, а не классического CEO.

**Где не сработало:** не было случаев, когда ИНН не нашёлся; но ФИО ЛПР по продажам/маркетингу из ЕГРЮЛ принципиально не достать.

### H3. zakupki.gov.ru / Контур.Закупки — контактные лица в тендерной документации

**Шаги:** Google-поиск «<ИНН> zakupki.gov.ru», переход на карточку заказчика. Также Контур.Закупки даёт сводки по ИНН без логина.

**Просмотрено записей:** для государственных и АО — Загорские дали (ФГБУ), Янган-Тау (АО госсобственность), Полтава-Крым (АО). Машук и Знание — частные ООО, в zakupki их нет как заказчиков. Реально перешёл на карточку Янган-Тау через Контур.Закупки (получил 403 при прямом fetch — публичная страница есть, но рендерится JS).

**Yield по выборке:** 0/5 — за этот проход не удалось вытащить конкретное контактное лицо из тендера. Это методологическое ограничение текущего прогона: чтобы получить контактное лицо, нужно открыть конкретную закупку и прочитать поле «Контактное лицо заказчика» — а Google по дефолту его не индексирует, нужно открывать сами карточки.

**Концептуально (опыт работы с zakupki ранее):** для гос-санаториев в карточке каждой опубликованной закупки есть ФИО+email+телефон контрактного управляющего. Это даёт стабильный канал входа в гос-санаторий через отдел закупок (хотя это не ЛПР по продажам). Yield концептуально оценивается как 3/5 для нашей выборки (государственные — Загорские дали, Янган-Тау, Полтава-Крым).

**Типы получаемых контактов:** ФИО контрактного управляющего, email (часто корпоративный), телефон, иногда руководитель контрактной службы.

**Ограничения:**
- Частные ООО (Машук, Знание) не размещают закупки на zakupki.gov.ru.
- Контрактный управляющий — это не ЛПР по решению о покупке услуг типа AI-агента; это, скорее, fallback на «как технически провести договор».
- Карточки закупок — динамические страницы, которые требуют либо ручного открытия, либо API/Контур.Закупки.

**Где не сработало:** на этом проходе — везде, по причине выше. В реальном проекте этот канал нужно проходить вручную после получения ИНН (через H2).

### H4. PDF на сайтах (приказы, прайсы, презентации)

**Шаги:** Google-запросы вида `site:<domain> filetype:pdf приказ|прайс|политика`. Также прямой обход разделов «Документы» на офсайте.

**Просмотрено записей:** 5 сайтов, ~10 поисковых запросов с filetype:pdf.

**Yield:** 1/5 — только Янган-Тау публикует структурированные PDF (Information disclosure, схема проезда). Эти PDF не содержат подписей с ФИО ЛПР.

**Типы получаемых контактов:** в принципе — подписи под приказами (ФИО+должность), email на бланках. На практике в этой выборке — почти ничего.

**Ограничения:**
- Большинство санаториев публикует прайсы в HTML, а не PDF.
- Приказы и распоряжения — внутренние документы, на офсайт не попадают.
- Презентации b2b — отправляются клиентам по запросу, а не выкладываются на сайте.
- Без OCR PDF-сканов работать тяжело: то, что найдётся, часто отсканировано как картинка.

**Где не сработало:** Машук, Знание, Загорские дали, Полтава-Крым — Google по `filetype:pdf` ничего по делу не вернул.

**Yield по «личным» аккаунтам ЛПР:** 0/5 — ни на одного директора не вышел через соцсети по прямому поиску. Это ожидаемо: директора 50+ лет в санаторно-курортной отрасли в РФ публично активны крайне редко, особенно не на стороне «инфлюэнсер-маркетинга».

**Типы получаемых контактов:** общий канал санатория (массовая аудитория, не ЛПР); иногда — ФИО маркетолога, ведущего соцсеть, в подписи поста (низкая частота).

**Где не сработало:** Полтава-Крым — соцсетей не видно вообще. Все 5 санаториев — нет персональных аккаунтов директоров.

### H8. hh.ru / SuperJob — вакансии и контактные лица HR

**Шаги:** поиск страницы работодателя по названию. Анализ вакансий: указано ли контактное лицо, телефон HR, ФИО HR-менеджера.

**Просмотрено записей:** 5 санаториев, у всех есть страница работодателя на hh.ru.

**Yield:**
- Контактный телефон HR: 5/5 (есть у всех).
- ФИО HR-менеджера: 1/5 (Янган-Тау — Мулюкова Г.Т.).
- Email HR / отдела кадров: 1/5 (Янган-Тау — `zakupki.yantau@mail.ru` указан как канал для резюме).
- ФИО руководителей подразделений в вакансиях («подчинение Иванову И.И.»): 0/5 — в нашей выборке такого формата не было.

**Типы получаемых контактов:** прямой телефон отдела кадров (не ЛПР, но это «человек, отвечающий на звонок» — путь эскалации), редко — ФИО HR.

**Ограничения:**
- HR-контакт ≠ ЛПР по продажам. Может работать как первичная точка входа для нетворкинга («хотим сотрудничество, кому переадресовать?»), но конверсия низкая.
- Часто на hh.ru указан общий телефон ресепшена, дублирующий офсайт.

**Где не сработало:** ни у одного санатория, кроме Янган-Тау, не получилось извлечь именно ФИО HR.

**Yield:** 0/5 — не найдено публичных профилей ЛПР наших санаториев в LinkedIn.

**Типы получаемых контактов:** —

**Где не сработало:** Загорские дали — корпоративные адреса вынесены на бесплатные провайдеры, шаблоны бесполезны.

### H11. СМИ / новости / интервью / отраслевые издания

**Шаги:** Google-поиск «<санаторий> <директор> интервью» + проверка профильных источников: AiF, РБК, региональные СМИ, отраслевые издания (TravelLine, hotelawards, sanatoria.ru).

**Просмотрено записей:** ~10 поисковых запросов на 5 санаториев.

**Yield:**
- Интервью с подтверждением ЛПР: 3/5
  - Янган-Тау — интервью с Акбашевым А.Р. в АиФ Уфа (подтверждает должность).
  - Знание (Сочи) — интервью с Дмитриевым Александром (директор по развитию) на TravelLine Blog. Это **самый ценный источник** в нашей выборке: Дмитриев — реально активный коммерческий ЛПР, использует CRM/BI, идеально соответствует профилю целевого клиента AGIents.
  - Знание — РБК Юг старое интервью с Богдановым Д.В. (бывший директор, сейчас под уголовным делом — этот контакт нерелевантен, но проясняет историю компании).
- Машук — нашлось публичное упоминание Карловой М.Г. на отраслевом hotelawards (в контексте номинации), но не интервью.
- Загорские дали — не нашлось публичных интервью с Мачулой Г.Б. (что логично для УД Президента — закрытое ведомство).
- Полтава-Крым — нет публичных интервью с действующим/бывшим руководством.

**Типы получаемых контактов:** ФИО ЛПР с подтверждением должности и зоны ответственности, иногда — упоминание используемого софта (мощный сигнал для AGIents — кому продавать в первую очередь). Прямого email/телефона в интервью обычно нет.

**Ограничения:**
- Интервью — точечный источник, не масштабируемый напрямую. Нельзя «собрать всех директоров через интервью».
- Старые интервью могут вводить в заблуждение (бывшие директора).
- Покрытие у частных средних санаториев низкое; у госсанаториев — почти нулевое.

**Где не сработало:** Загорские дали, Полтава-Крым — нет интервью.

## 4. Рекомендации по автоматизации

### 4.1. Что автоматизировать в первую очередь (high ROI, низкая сложность)

**Слой 1: «Карточка санатория»** — для каждого санатория собирать сразу 3 источника:

1. **Парсинг офсайта (H1):** `requests` + `BeautifulSoup` + selenium для тяжёлых JS-страниц.
   - Целевые страницы: `/kontakty/`, `/contacts/`, `/о-санатории/`, `/руководство/`, `/документы/`, `/karera/`.
   - Регулярки на email (`[\w\.-]+@[\w\.-]+\.\w+`) и телефоны (`\+?7[\s\-\(]?\d{3,4}[\)\-\s]?\d{3}[\s\-]?\d{2}[\s\-]?\d{2}`).
   - LLM-постобработка для извлечения ФИО+должность из секций «Руководство»/«Наши специалисты» (это структурированные блоки, на которых GPT-4-mini/Haiku 4.5 работает с >95% точностью при правильном промпте).
   - Стоимость: ~30–60 сек/санаторий, $0.001–0.003 на LLM.

2. **RusProfile/list-org/checko/saby (H2):** парсинг публичной части по ИНН/названию.
   - **Важно:** RusProfile активно ставит anti-bot. Лучше использовать checko.ru или list-org.com (мягче), либо интегрировать DaData (платно, но дёшево — ~30 коп/запрос за реквизиты).
   - Извлекать: ИНН, ОГРН, ФИО руководителя+должность+дату назначения, юр. адрес, учредителей, ОКВЭД.
   - Стоимость через DaData: ~$0.005/санаторий.

3. **Email pattern enrichment (H10):** для каждого домена санатория генерировать набор шаблонных адресов и валидировать через SMTP / `mailboxlayer` / `Hunter.io free tier`.
   - Шаблоны (отсортированы по подтверждённой частоте в этой выборке): `info@`, `sales@`, `marketing@`, `market@`, `b2b@`, `secr@`, `konsultant@`, `director@`, `<имя.фамилия>@` (после получения ФИО из H2).
   - Валидация: SMTP-ping без отправки письма, либо Hunter.io (50 free/мес).
   - Стоимость: бесплатно для small batch; при масштабе — ~$0.01/проверка.

**Связка слоёв 1+2+3 покрывает ~70–80% задачи** «получить контакт для входа в санаторий» без платных enrichment.

**Слой 2: «Сигналы зрелости» (доп. сегментация для приоритизации):**

4. **Парсинг hh.ru работодателя (H8):** активность (количество вакансий за 6/12 мес), типы открытых позиций (если ищут «директор по маркетингу» / «руководитель отдела продаж» — это горячий сигнал зрелости).
   - hh.ru имеет официальный API, но он закрытый; реально — парсинг страницы employer.
   - Стоимость: ~2 сек/санаторий, бесплатно.

5. **Соцсети — наличие и базовая активность (H5):** через VK API (бесплатно, требует токен) + проверка t.me/`<handle>`.
   - Метрики: подписчики, частота постов, упоминания ФИО в постах.
   - Стоимость: бесплатно через VK API, ~5 сек/санаторий.

### 4.2. Что оставить ручным (низкий yield автоматизации, но высокая ценность для ключевых аккаунтов)

6. **СМИ / отраслевые интервью (H11):** ручной поиск перед outreach по ключевым санаториям.
   - Это даёт **самый ценный сигнал**: упомянутый в интервью ЛПР по развитию (как Дмитриев из Знания) — это уже почти-квалифицированный лид, потому что он публично говорит про автоматизацию/CRM/BI.
   - Автоматизировать через LLM-агента можно, но качество будет хуже ручного: интервью смешаны со SEO-мусором, нужна квалифицированная фильтрация.
   - **Решение:** ручной подготовительный ресёрч на топ-20 санаториев (час времени) → выкатывать персонализированный outreach.

7. **PDF-документы (H4):** низкий yield в санаторно-курортной отрасли. Не автоматизировать в первой версии.
   - Если санаторий гос/АО — иногда PDF с приказами лежат в разделе «Раскрытие информации» (как у Янган-Тау). Точечная проверка на топ-санаториях вручную.

8. **Запись разговора с приёмной директора:** для топ-аккаунтов — ручной звонок секретарю с просьбой «соедините с директором по развитию / маркетингу». В РФ-санаториях это часто работает лучше cold email; стоит держать как параллельный канал.

9. **Подтверждение актуальности ФИО директора:** ЕГРЮЛ обновляется с задержкой; если ФИО/должность критична для outreach — ручная сверка через 2-3 источника (rusprofile + офсайт + интервью или новости).

### 4.3. Где подключать платный enrichment (по этапам воронки)

**Этап 1 — Reference data (массово, дёшево):**
- **DaData (ИП/ООО реквизиты):** ~30 коп/запрос. Покрывает H2 быстрее и стабильнее парсинга. Подключать сразу на стадии формирования базы санаториев.
- **Контур.Фокус API:** дороже, но богаче. Имеет смысл, если нужны фин. показатели (выручка, прибыль) для приоритизации — крупные санатории = больший чек.

**Этап 2 — Email validation:**
- **Hunter.io:** ~$0.01/lookup, домен-pattern + valid emails. Отлично для масштабирования H10.
- **Snov.io:** альтернатива Hunter, дешевле в РФ-сегменте.

**Этап 3 — ФИО + должности других ЛПР (директор по маркетингу, ком. директор):**
- **HuntIO / Lusha / Apollo.io:** западные базы; для РФ-санаториев покрытие 5–15% (основано на низкой представленности в LinkedIn). Цена не оправдана для этой ниши.
- **LinkedIn Sales Navigator:** для РФ-санаториев — почти не работает (см. H9 yield = 0). Не подключать.
- **Российские альтернативы:**
  - **HRDirector-базы / Мегаплан-базы / Работа.ру для b2b** — точечные, дорогие, качество разное.
  - **FindyMail / Skylead** — частично работают на доменах @yandex.ru / @mail.ru, что характерно для гос-санаториев.
- **Важный вывод:** платный enrichment для ЛПР конкретно по санаторно-курортной отрасли РФ имеет ограниченную ценность. Лучшее соотношение цена/качество — **DaData (формальный CEO) + Hunter (общие email-ы) + ручной звонок приёмной**.

**Этап 4 — История взаимодействий и интент:**
- **DaData по тендерам / Контур.Закупки API:** даёт сигнал «что закупает», что вообще закупает (если ничего — частник, если IT-системы — горячий клиент).
- Это уже более продвинутая сегментация; имеет смысл на 2-й итерации.

### 4.4. Приоритезация для AGIents

**Версия v1 (MVP пайплайн поиска контактов):**
1. Список санаториев (от 100 до 1000) с ИНН.
2. DaData → формальный CEO + реквизиты.
3. Парсинг офсайта → email отделов + соцсети + ФИО мед-руководства (если есть).
4. Hunter.io → валидация сгенерированных шаблонных email-ов.
5. Сегментация по сигналам зрелости: активность hh.ru + наличие активных соцсетей + отраслевые интервью (ручной чекпоинт на топ-N).
6. Outreach: один email на `marketing@/sales@/b2b@` + копия на CEO email (если найден через Hunter).

**Версия v2 (улучшения):**
- Интеграция Контур.Закупки API для сигналов о реальной закупочной активности.
- LLM-агент для парсинга интервью / отраслевых СМИ и сборки «карточки активного ЛПР» по топу.
- Звонок-бот (с человеческим оператором) для эскалации топ-аккаунтов через приёмную.

**Что НЕ строить:**
- Свой LinkedIn-парсер (нелегально + не работает в этой нише).
- OCR-пайплайн на PDF санаториев (yield слишком низкий для ROI).
- Автоматизацию TikTok/Instagram-парсинга для санаторных директоров (их там нет).

## 5. Ограничения и оговорки

- **Размер выборки 5** — этого достаточно, чтобы выявить структурные особенности отрасли (где офсайт молчит, где ЕГРЮЛ работает, где помогают интервью), но мало для статистических выводов вида «X% санаториев публикуют ФИО гендира». Для этого нужна выборка ~100+.
- **Один проход H3 (zakupki) — концептуальный.** Реальный yield из тендерной документации проверен на других проектах, но в этой сессии — ограничение по времени/глубине.
- **Динамика данных:** ФИО директоров меняются. На любую конкретную организацию надо проверять актуальность через 2-3 источника, особенно если последний апдейт ЕГРЮЛ старше 6 мес.
- **Не сфабриковано:** все ФИО, email-ы, телефоны и URL получены из открытых источников и в этом отчёте указаны источники для каждой записи. Записи с конфликтами (Машук — Боков vs Карлова; Полтава — Попов vs Ракоцило) явно помечены как `low confidence`.
