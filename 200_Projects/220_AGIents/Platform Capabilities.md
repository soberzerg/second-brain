# AGIents — Текущие возможности платформы

Снимок реализованного функционала AGIents на 2026-04-20. Используется как справочник при скоупинге клиентских проектов (например, [[Discovery Call 2026-04-16|Krytex]]) — чтобы не переоценивать объём работы и отличать "уже есть" от "надо делать".

## Уже реализовано

### Inbox диалогов

- Все переписки бота с клиентами в едином интерфейсе
- Фильтрация по каналу (Telegram, VK, MAX и др.)
- Поиск по содержимому диалогов
- Просмотр контекста: кто клиент, когда писал, как бот отвечал

### Аналитика

- Счётчики входящих вопросов (день/неделя/месяц)
- % автономных ответов vs escalations
- Топ-вопросы без ответа (gap в базе знаний)
- Расход токенов LLM

### Настройки бота

- Редактирование системного промпта
- Confidence-порог (при каком уровне уверенности отвечать автономно, при каком эскалировать)
- Режим работы: auto / HITL / гибрид
- Расписание работы (рабочие часы / 24×7)

### Управление пользователями

- Приглашение менеджеров со стороны клиента
- Роли: админ / менеджер / читатель
- Разграничение доступа к функциям

### Сессионная память диалога

- Бот помнит последние N сообщений в рамках активной сессии
- Сессия закрывается по таймауту неактивности
- История между сессиями не сохраняется — экономия токенов
- Позволяет вести естественный многошаговый диалог ("а для чего он?" — бот помнит, о каком продукте речь)

### Интеграция с CRM (amoCRM и аналоги)

- Двусторонняя интеграция через API: чтение и запись контактов, сделок, задач
- Идентификация клиента по номеру телефона: поиск карточки контакта, определение типа обращения (новый / повторный / текущий)
- Извлечение контекста из CRM: имя, отчество, заметки, история сделок, активные сделки на разных этапах воронки
- Чтение параметров сделки: текущий этап, даты бронирования/услуги, категория, сумма, состав участников
- Регрессионная защита логики дедупликации контактов на стороне CRM

### Управление воронкой продаж

- Бот ведёт клиента по настраиваемой воронке: переводит сделки между этапами по правилам сценария
- Сбор параметров сделки в диалоге (даты, категории, количество участников, персональные данные)
- Формирование и отправка ссылки на оплату, перевод сделки на этап предоплаты
- Этапы с активным управлением и этапы «только консультация» настраиваются отдельно
- Поддержка особых сценариев на терминальных этапах (отзыв / повторное обращение / новая сделка)
- Защитные ограничения: запрещённые переходы (например, перевод в «Отказ») выполняются только менеджером

### Эскалация диалога менеджеру

- Настраиваемые триггеры эскалации: низкий confidence, отсутствие ответа в базе знаний, ключевые слова, бизнес-правила (лимиты по количеству участников, особые категории клиентов и т.п.)
- Постановка задачи менеджеру в CRM с типизацией (например, тип «WhatsApp») и полным контекстом диалога
- Распределение задач — по существующей логике CRM, без дублирования
- Авто-уведомление клиента: в рабочее время — о подключении менеджера, в нерабочее — о времени ответа
- Раздельная логика рабочего и нерабочего времени с продолжением консультаций ботом 24×7

### Переключение бот ↔ менеджер

- Авто-пауза бота при входе менеджера в сделку — бот не вмешивается, пока менеджер ведёт диалог
- Watchdog: если менеджер не отвечает заданный таймаут (например, 20 минут) — бот возвращается к диалогу или ставит задачу
- Авто-возврат диалога боту при новом входящем сообщении от клиента
- Гибкая пауза при возврате: учитывается, общался ли менеджер с клиентом сегодня

### Финальное касание и реактивация

- Отслеживание «зависших» сделок: бот отправляет ситуативное follow-up сообщение по таймеру (например, 5 дней без активности)
- Если клиент не отвечает на финальное касание — задача менеджеру для принятия решения (без авто-перевода в «Отказ»)
- Настраиваемые шаблоны сообщений под этап воронки

### Динамическое расписание по графику команды

- Определение рабочего/нерабочего времени по фактическому графику менеджеров, а не по фиксированным часам
- Учёт смен, выходных, праздников, индивидуальных графиков
- Авто-переключение режимов: если ни один менеджер не на смене — бот работает в режиме «нерабочее время»
- Динамическое обновление графика без перезапуска

### Сбор данных и оформление документов

- Структурированный сбор данных у клиента по сценарию (например, для налогового вычета или оформления документов)
- Передача собранных данных менеджеру задачей в CRM
- Валидация полей на стороне бота перед эскалацией

### Оценка ответов бота менеджерами (HITL feedback loop)

- Сообщения бота визуально отличаются от сообщений менеджеров в CRM
- Иконки оценки «палец вверх» / «палец вниз» у каждого ответа бота
- Поле для комментария при негативной оценке
- Все оценки и комментарии сохраняются и доступны для анализа и дообучения

### Карта лояльности и промокоды

- Единый идентификатор клиента (например, номер телефона) как промокод и номер карты лояльности
- Автоматическое применение промокода при наличии телефона клиента (из канала или собранного в диалоге)
- Накопление и использование бонусов в сценариях бронирования

### Расширенная аналитика и дашборды

- Операционные метрики: количество диалогов, доля автономных, эскалации по типам триггеров, время ответа бота и реакции менеджера
- Конверсионные метрики: воронка ведения сделок ботом, конверсия по этапам, сравнение бот vs менеджер, средний чек
- Метрики качества: распределение оценок менеджеров, частота негативных оценок по типам вопросов, результативность финального касания
- Экономические метрики: расчётная экономия рабочего времени, выручка по сделкам с участием бота, стоимость обработки одного обращения, обращения в нерабочее время
- Дашборд в реальном времени + еженедельные/ежемесячные авто-отчёты
- Фильтрация по периоду, этапу воронки, времени суток, каналу

## Ещё не реализовано (требует разработки под проект)

### Редактор базы знаний (Q&A)

- Веб-интерфейс для CRUD по таблице Q&A
- Поиск, теги, категории
- История изменений, откат

### Лог escalations

- Отдельная вьюха — только диалоги, где бот не справился
- Контекст: вопрос клиента + предложенный ботом ответ + причина эскалации (низкий confidence / явная просьба человека / ключевое слово)
- Ответ менеджера из этой вьюхи → возврат клиенту + опционально сохранение в базу знаний

### Менеджерский pipeline для маркетплейсов

- Отдельный бот в Telegram для менеджеров
- Входящие Q&A с Ozon/WB через API (read)
- Черновик ответа от бота → менеджер использует в ручной публикации
- Бот не пишет в маркетплейсы сам

### Мультиканальная доставка в новые каналы

- VK и MAX — требуют интеграции (Telegram уже готов)
- Виджет на сайте, Instagram, WhatsApp — фаза 2+
