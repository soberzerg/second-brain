# PRD: Мультиагентная AI-система юридических консультаций Dogovor24

> Коммерческое предложение AGIency для Dogovor24 — интеллектуальная система ответов на юридические вопросы пользователей dogovor24.kz

## 1. Обзор проекта

### Проблема

Платформа dogovor24.kz получает устойчивый поток юридических вопросов от пользователей. Сейчас вопросы обрабатываются вручную через внутренний сервис **amicus**: модератор принимает вопрос, классифицирует, передаёт юристу соответствующей специализации, юрист готовит ответ после оплаты услуги. Это создаёт системные ограничения:

- **Высокая нагрузка на юристов:** даже типовые вопросы (которые повторяются регулярно) требуют ручной обработки
- **Долгое время ответа:** пользователь ждёт, пока вопрос пройдёт через модерацию и дойдёт до юриста
- **Ограниченная масштабируемость:** количество обработанных вопросов прямо пропорционально количеству юристов
- **Неоднородное качество:** разные юристы дают ответы разного качества и детализации на одинаковые вопросы
- **Потеря конверсии:** пользователи уходят, не дождавшись ответа, или не готовы платить за консультацию по простому вопросу

### Решение

Мультиагентная AI-система с роутером, которая:

- Принимает юридический вопрос пользователя
- Классифицирует область права и сложность вопроса
- Маршрутизирует на специализированного AI-агента (или группу агентов)
- Генерирует юридически обоснованный ответ с опорой на НПА РК
- Передаёт ответ на модерацию человеку (Phase 1) или отправляет автоматически (Phase 3)
- Эскалирует на юриста-человека, если AI не может дать качественный ответ

### Целевой результат

Сократить время ответа на типовые юридические вопросы с **часов/дней до минут**, снизить нагрузку на юристов на **60-80%** для типовых вопросов, повысить конверсию пользователей за счёт мгновенной первичной консультации.

## 2. Контекст: существующая инфраструктура

### Платформа Dogovor24

| Параметр | Значение |
|----------|----------|
| Профиль | Multi-tenant платформа электронного документооборота и цифровой подписи (Казахстан, Россия) |

| API-паттерн | REST API с префиксом `/api/amicus` |
| Асинхронные коммуникации | Event-driven архитектура (RabbitMQ/Redis) |
| Аутентификация | JWT-токены через authorization-service |

### Сервис Amicus — текущий workflow

```
Пользователь (dogovor24.kz)
       │
       ▼
  Задаёт вопрос юристу
       │
       ▼
  amicus-service: приём вопроса
       │
       ▼
  Модератор: классификация + проверка
       │
       ▼
  Передача юристу соответствующей специализации
       │
       ▼
  Пользователь: оплата услуги
       │
       ▼
  Юрист: подготовка ответа
       │
       ▼
  Пользователь: получение ответа
```

### 3.1 Общая схема мультиагентной системы

```
Пользователь (dogovor24.kz)
       │
       ▼
  amicus-service (существующий)
       │
       ▼ API-вызов

  │          AI Legal Assistant Gateway         │
  │                                            │
  │  ┌──────────────────────────────────────┐  │
  │  │          Router Agent                 │  │

  │  │  - Оценка сложности (simple/complex) │  │
  │  │  - Маршрутизация                     │  │
  │  └───────────┬──────────────────────────┘  │
  │              │                              │
  │    ┌─────────┼─────────┬──────────┐        │
  │    ▼         ▼         ▼          ▼        │
  │ ┌──────┐ ┌──────┐ ┌──────┐  ┌─────────┐   │
  │ │Трудо-│ │Граж- │ │Семей-│  │Корпора- │   │
  │ │вое   │ │дан-  │ │ное   │  │тивное   │   │

  │ └──┬───┘ └──┬───┘ └──┬───┘  └────┬────┘   │
  │    └─────────┴────────┴───────────┘        │
  │              │                              │
  │              ▼                              │
  │  ┌──────────────────────────────────────┐  │
  │  │       Response Composer Agent         │  │
  │  │  - Сборка ответа + ссылки на НПА     │  │
  │  │  - Дисклеймер                        │  │

### 3.2 Компоненты системы

| Компонент | Тип | Описание |
|-----------|-----|----------|
| **AI Legal Gateway** | Новый сервис | API-шлюз между amicus-service и мультиагентной системой |
| **Router Agent** | AI-агент | Классификация вопроса, определение области права, оценка сложности, маршрутизация |
| **Специализированные агенты** | AI-агенты | Агенты по областям права с доменными знаниями и релевантной базой НПА |
| **Response Composer** | AI-агент | Финальная сборка ответа, добавление ссылок на НПА, формирование дисклеймера |
| **Knowledge Base (НПА РК)** | RAG-хранилище | Векторная база нормативно-правовых актов Республики Казахстан |
| **Moderation Queue** | UI + API | Очередь модерации AI-ответов для юристов-проверяющих |
| **Analytics & Feedback** | Сервис | Сбор метрик качества, обратной связи, обучение системы |

### 3.3 Router Agent — логика маршрутизации

Router Agent решает три задачи:

**1. Классификация области права:**

| Область | Примеры вопросов | Специализированный агент |
|---------|-----------------|-------------------------|
| Трудовое право | Увольнение, отпуск, зарплата, трудовой договор | `labor-law-agent` |
| Гражданское право | Договоры, обязательства, наследство, собственность | `civil-law-agent` |
| Семейное право | Развод, алименты, раздел имущества, опека | `family-law-agent` |
| Корпоративное право | Регистрация ТОО/ИП, учредительные документы, ликвидация | `corporate-law-agent` |
| Жилищное право | Аренда, покупка/продажа недвижимости, ЖКХ, приватизация | `housing-law-agent` |
| Административное право | Штрафы, разрешения, лицензии, гос. услуги | `admin-law-agent` |
| Налоговое право | Налоги ИП/ТОО, декларации, проверки | `tax-law-agent` |
| Земельное право | Участки, целевое назначение, кадастр | `land-law-agent` |

**2. Оценка сложности:**

| Уровень | Критерии | Действие |
|---------|----------|----------|
| **Simple** | Типовой вопрос, однозначный ответ по НПА, высокая частотность | AI отвечает самостоятельно |
| **Medium** | Требует анализа нескольких НПА, контекстно-зависимый ответ | AI отвечает с повышенным порогом модерации |
| **Complex** | Неоднозначная правоприменительная практика, требуется анализ судебных решений | Эскалация на юриста |
| **Out of scope** | Не юридический вопрос, другая юрисдикция, уголовное право | Перенаправление или отказ с пояснением |

**3. Мультидоменная маршрутизация:**

Если вопрос затрагивает несколько областей права (например, «раздел бизнеса при разводе» — семейное + корпоративное), Router Agent запускает параллельно несколько специализированных агентов и передаёт результаты в Response Composer для синтеза.

### 3.4 Специализированные агенты — устройство

Каждый специализированный агент состоит из:

```
┌─────────────────────────────────────────┐

│  ┌─────────────────────────────────┐    │
│  │  System Prompt                   │    │
│  │  - Роль: юрист-специалист       │    │

### 3.5 Knowledge Base (НПА РК)

**Источники нормативных актов:**

| Источник | Тип | Приоритет |
|----------|-----|-----------|
| [adilet.zan.kz](https://adilet.zan.kz) | Официальная база НПА РК | Основной |
| Кодексы РК | Трудовой, Гражданский, Налоговый, Земельный и др. | Основной |
| Законы РК | Профильные законы по областям права | Основной |
| Нормативные постановления Верховного суда | Разъяснения правоприменения | Дополнительный |
| Подзаконные акты | Приказы, постановления правительства | Дополнительный |

**Технические требования к Knowledge Base:**

- Векторная база данных (pgvector / Qdrant / Pinecone) для семантического поиска
- Чанкинг документов с сохранением структуры (статья → пункт → подпункт)
- Метаданные: номер НПА, дата принятия, дата последних изменений, область права, статус (действующий/утратил силу)
- Регулярное обновление: проверка изменений в НПА минимум 1 раз в неделю
- Гибридный поиск: semantic (embeddings) + keyword (BM25) для максимальной точности

### 3.6 Response Composer — формат ответа

Каждый AI-ответ пользователю содержит:

```
1. Краткий ответ (1-2 предложения)

2. Развёрнутое обоснование
   - Ссылки на конкретные статьи НПА
   - Пошаговые действия (если применимо)

3. Ссылки на НПА
   - [Трудовой кодекс РК, ст. 53] — точная ссылка
   - [Закон РК "О ...", ст. 15, п. 3] — точная ссылка

4. Дисклеймер
   "Данный ответ подготовлен AI-ассистентом на основе действующего
   законодательства РК и носит информационно-справочный характер.
   Для получения юридически обязывающей консультации по вашей
   конкретной ситуации рекомендуем обратиться к юристу."

5. Confidence score (внутренний, не показывается пользователю)
```

## 4. Интеграция с amicus-service

### 4.1 API-контракт

AI Legal Gateway предоставляет REST API для amicus-service:

**POST `/api/ai-legal/questions`** — отправка вопроса на AI-обработку

```json
{
  "question_id": "uuid",
  "user_id": "uuid",
  "question_text": "Могу ли я уволиться без отработки двух недель?",
  "category_hint": "labor",      // опциональная подсказка от amicus
  "context": {                   // опциональный контекст
    "region": "almaty",
    "user_type": "individual"    // individual | business
  }
}
```

**Response:**

```json
{
  "question_id": "uuid",
  "status": "answered | escalated | pending_moderation",
  "answer": {
    "summary": "...",
    "detailed": "...",
    "legal_references": [
      {
        "npa_title": "Трудовой кодекс РК",
        "article": "56",
        "paragraph": "1",
        "url": "https://adilet.zan.kz/..."
      }
    ],
    "disclaimer": "..."
  },
  "metadata": {
    "detected_category": "labor",
    "complexity": "simple",
    "confidence": 0.92,
    "agents_used": ["router", "labor-law-agent", "response-composer"],
    "processing_time_ms": 3200
  },
  "escalation": null   // или { "reason": "...", "recommended_specialist": "..." }
}
```

**GET `/api/ai-legal/questions/{id}/status`** — статус обработки

**POST `/api/ai-legal/moderation/{id}/approve`** — одобрение модератором
**POST `/api/ai-legal/moderation/{id}/reject`** — отклонение с правкой
**POST `/api/ai-legal/moderation/{id}/escalate`** — эскалация на юриста

### 4.2 Паттерн интеграции

```
amicus-service                    AI Legal Gateway
      │                                  │
      │── POST /questions ──────────────▶│
      │                                  │── Router Agent
      │                                  │── Specialized Agent(s)
      │                                  │── Response Composer
      │◀── 202 Accepted (async) ─────────│
      │                                  │
      │    [Webhook / Polling]           │
      │◀── POST /callback ──────────────│ (ответ готов)
      │                                  │
      │── [Phase 1: модерация] ─────────▶│
      │── POST /moderation/approve ─────▶│
      │                                  │
      ▼                                  │
  Ответ пользователю                     │
```

**Два режима интеграции:**

| Режим | Описание | Когда использовать |
|-------|----------|-------------------|
| **Синхронный** | amicus ждёт ответ (timeout 30s) | Phase 3, простые вопросы с высоким confidence |
| **Асинхронный** | amicus получает callback/webhook | Phase 1-2, все вопросы с модерацией |

## 5. Workflow модерации AI-ответов

### 5.1 Phase 1 — полная модерация

Все AI-ответы проходят через модератора перед отправкой пользователю.

```
AI генерирует ответ
       │
       ▼
  Ответ попадает в Moderation Queue
       │
       ▼
  Модератор видит:
  - Вопрос пользователя
  - AI-ответ (с подсветкой ссылок на НПА)
  - Confidence score
  - Определённая область права
  - Время генерации
       │
       ├── [Approve] → Ответ отправляется пользователю
       │
       ├── [Edit + Approve] → Модератор корректирует → отправка
       │                      (правки сохраняются для обучения)
       │
       ├── [Reject] → Ответ отклоняется
       │              (причина отклонения сохраняется для обучения)
       │
       └── [Escalate] → Вопрос передаётся юристу-человеку
                        (через стандартный amicus workflow)
```

### 5.2 UI модерации — требования

Интерфейс модерации встраивается в существующий **amicus-crm** (Vue.js/Nuxt.js):

- Очередь ответов с сортировкой по приоритету (низкий confidence первым)
- Diff-view: показ AI-ответа рядом с вопросом
- Inline-редактирование ответа
- Подсветка ссылок на НПА с возможностью кликнуть и проверить
- Статистика модератора: количество обработанных, среднее время, процент approve/reject
- Горячие клавиши для быстрой модерации (Approve/Reject одним нажатием)

### 5.3 Обратная связь для обучения

Каждое действие модератора сохраняется как обучающий сигнал:

| Действие | Что сохраняется | Как используется |
|----------|-----------------|------------------|
| Approve | Пара вопрос-ответ как эталон | Fine-tuning, few-shot примеры |
| Edit + Approve | Оригинал + правка | Обучение на ошибках, корректировка промптов |
| Reject | Причина отклонения | Выявление систематических ошибок |
| Escalate | Причина эскалации | Расширение правил эскалации |

## 6. Механизм эскалации на юриста

### 6.1 Автоматическая эскалация (AI принимает решение)

AI эскалирует вопрос на юриста-человека, если:

| Условие | Порог | Пример |
|---------|-------|--------|
| Низкий confidence | < 0.6 | Неоднозначный вопрос, нет релевантных НПА |
| Высокая сложность | complexity = "complex" | Требует анализа судебной практики |
| Конфликт НПА | Противоречащие нормы | Коллизия между кодексом и подзаконным актом |
| Вне покрытия | out_of_scope | Уголовное право, другая юрисдикция |
| Персональные данные | PII detected | Вопрос содержит чувствительные данные, требующие индивидуального подхода |
| Эмоциональный стресс | sentiment = distress | Пользователь в явно тяжёлой ситуации |

### 6.2 Ручная эскалация (модератор принимает решение)

Модератор эскалирует, если AI-ответ:

- Фактически неверен
- Неполон (пропущены важные аспекты)
- Потенциально вреден (неправильный совет может привести к убыткам)
- Требует индивидуального анализа документов клиента

### 6.3 Процесс эскалации

```
AI / Модератор: решение об эскалации
       │
       ▼
  Формируется пакет эскалации:
  - Оригинальный вопрос пользователя
  - AI-анализ (что удалось определить)
  - Определённая область права
  - Причина эскалации
  - Релевантные НПА (найденные AI)
       │
       ▼
  Передача в amicus-service
  (стандартный workflow назначения юриста)
       │
       ▼
  Юрист получает вопрос + AI-подготовку
  (AI экономит время юриста на первичный анализ)
       │
       ▼
  Пользователю: уведомление
  "Ваш вопрос требует детального анализа юристом.
   Вам ответит специалист по {область}."
```

## 7. Стратегия перехода от модерации к автоматизации

### 7.1 Модель зрелости

| Phase | Модерация | Автоматика | Критерии перехода |
|-------|-----------|------------|-------------------|
| **Phase 1** | 100% ответов проходят модерацию | 0% автоматических ответов | Запуск системы |
| **Phase 2a** | Ответы с confidence < 0.85 | Автоматика для simple + confidence >= 0.85 | Approve rate > 90% на 500+ ответах |
| **Phase 2b** | Ответы с confidence < 0.75 | Автоматика для simple/medium + confidence >= 0.75 | Approve rate > 95% на 2000+ ответах |
| **Phase 3** | Только complex и edge cases | Автоматика для всех ответов с confidence >= 0.70 | Approve rate > 97% на 5000+ ответах, 0 критических ошибок за 30 дней |

### 7.2 Механизм автоматического повышения доверия

```
Для каждой пары (область_права, тип_сложности):
  1. Считаем approve_rate за последние N ответов
  2. Если approve_rate >= порог И N >= минимальная выборка:
     → Переводим эту пару в автоматический режим
  3. Если после перехода обнаружена ошибка:
     → Откатываем эту пару обратно в режим модерации
     → Анализируем причину, корректируем агента
     → Повторяем набор статистики
```

### 7.3 Spot-check модерация (Phase 3)

Даже в полностью автоматическом режиме:

- **5-10% ответов** случайным образом проверяются модератором
- Все ответы с confidence в «серой зоне» (0.70-0.80) проверяются
- Еженедельный аудит: выборочная проверка 50 автоматических ответов
- Любая обнаруженная ошибка немедленно откатывает соответствующую категорию в режим модерации

## 9. Функциональные требования

### 9.1 Must Have (P0)

- Как **пользователь dogovor24.kz**, я хочу задать юридический вопрос на русском языке и получить ответ с обоснованием по НПА РК, чтобы понять свои права и следующие шаги
- Как **пользователь**, я хочу видеть ссылки на конкретные статьи законов в ответе, чтобы проверить информацию самостоятельно
- Как **пользователь**, я хочу быть уведомлен, если мой вопрос передан юристу-человеку, чтобы понимать статус обработки
- Как **модератор amicus**, я хочу видеть AI-ответы в очереди модерации и одобрять/отклонять/редактировать их, чтобы контролировать качество
- Как **модератор**, я хочу видеть confidence score и определённую область права, чтобы быстрее оценить качество AI-ответа
- Как **администратор**, я хочу видеть метрики работы системы (approve rate, время ответа, процент эскалаций), чтобы оценивать эффективность
- Как **система**, Router Agent классифицирует вопрос по области права с точностью не ниже 90%
- Как **система**, при confidence < 0.6 вопрос автоматически эскалируется на юриста

### 9.2 Should Have (P1)

- Как **пользователь**, я хочу задавать вопросы на казахском языке, чтобы получить ответ на родном языке
- Как **пользователь**, я хочу задать уточняющий вопрос к полученному ответу (follow-up), чтобы разобраться в деталях
- Как **модератор**, я хочу использовать горячие клавиши для быстрой модерации, чтобы обрабатывать больше ответов
- Как **модератор**, я хочу видеть похожие ранее одобренные ответы (для сравнения), чтобы ускорить проверку
- Как **юрист**, при эскалации я хочу видеть AI-подготовку (определённая область, найденные НПА, предварительный анализ), чтобы быстрее подготовить ответ
- Как **администратор**, я хочу настраивать пороги confidence для автоматической отправки/эскалации, чтобы гибко управлять балансом скорость/качество

### 9.3 Nice to Have (P2)

- Как **пользователь**, я хочу получить ответ в виде пошаговой инструкции (checklist), чтобы точно знать, какие документы и действия нужны
- Как **пользователь**, я хочу прикрепить документ (фото/скан) к вопросу, чтобы AI учёл его содержание
- Как **администратор**, я хочу A/B-тестировать промпты специализированных агентов, чтобы улучшать качество
- Как **система**, AI предлагает релевантные платные услуги на dogovor24.kz (шаблоны документов, юридические услуги) в конце ответа
- Как **система**, AI определяет типовые вопросы и предлагает FAQ-ответы без запуска мультиагентного pipeline

## 11. Метрики успеха

### 11.1 Ключевые метрики

| Метрика | Определение | Целевое значение |
|---------|------------|------------------|
| **Approve Rate** | % AI-ответов, одобренных модератором без правок | > 85% (Phase 1), > 95% (Phase 3) |
| **Время до ответа** | От вопроса до получения ответа пользователем | < 10 мин (с модерацией), < 1 мин (без) |
| **Escalation Rate** | % вопросов, эскалированных на юриста | < 25% (Phase 1), < 10% (Phase 3) |
| **Classification Accuracy** | Точность определения области права | > 90% |
| **User Satisfaction** | Оценка ответа пользователем (1-5) | > 4.0 |
| **Cost per Answer** | Стоимость одного AI-ответа (LLM + инфраструктура) | < 50% стоимости ответа юриста |
| **Юрист Productivity Lift** | Рост количества обработанных вопросов на юриста | +50% (за счёт AI-подготовки при эскалации) |

### 11.2 Операционные метрики

| Метрика | Описание |
|---------|----------|
| Среднее время модерации | Сколько секунд модератор тратит на один ответ |
| Queue depth | Размер очереди модерации (не должна расти) |
| Edit rate | % ответов, потребовавших правки модератором |
| Rejection rate | % отклонённых ответов (маркер системных проблем) |
| НПА coverage | % ответов с корректными ссылками на НПА |
| Agent latency p95 | 95-й перцентиль времени работы каждого агента |

## 12. Non-Goals (вне скопа)

Система **НЕ будет**:

- Давать ответы по уголовному праву (слишком высокие риски, требуется адвокат)
- Давать ответы по законодательству других стран (только НПА РК)
- Составлять юридические документы (это отдельный сервис dogovor24.kz)
- Представлять пользователя в суде или давать процессуальные советы
- Заменять юристов-людей для сложных вопросов
- Работать как chatbot с длинной историей диалога (на Phase 1 — формат "вопрос-ответ")
- Обрабатывать вопросы, требующие анализа конфиденциальных документов (на Phase 1)
- Интегрироваться с внешними CRM-системами (только amicus-service)

## 13. Риски и митигация

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|

| **Устаревшие НПА в Knowledge Base** | Средняя | Высокое | Автоматическое обновление; метаданные с датой актуальности; уведомления об изменениях |
| **Низкий approve rate — модераторы перегружены** | Средняя | Среднее | Итеративное улучшение промптов; расширение few-shot примеров; A/B-тестирование |
| **Пользователи воспринимают AI-ответ как юридически обязывающий** | Высокая | Высокое | Дисклеймер в каждом ответе; UI-разделение AI и юриста; обучающие подсказки |
| **Недостаточное покрытие НПА для отдельных областей права** | Средняя | Среднее | Phased rollout по областям; начать с наиболее покрытых областей (трудовое, гражданское) |
| **Высокая стоимость LLM-вызовов при масштабировании** | Средняя | Среднее | Кэширование типовых вопросов; использование более дешёвых моделей для simple-вопросов; prompt optimization |
| **Регуляторные риски (персональные данные, ответственность за AI-совет)** | Низкая | Высокое | Консультация с юридическим отделом D24; соблюдение Закона РК о персональных данных; разграничение «справочная информация» vs «юридическая консультация» |
| **Сопротивление юристов (страх замены)** | Средняя | Среднее | Позиционирование AI как помощника, а не замены; демонстрация роста productivity; сохранение сложных вопросов за людьми |

- Соблюдение Закона РК «О персональных данных и их защите»

## 15. Этапы внедрения

### Phase 1: MVP — полная модерация (8-10 недель)

| Неделя | Задача |
|--------|--------|
| 1-2 | Проектирование архитектуры, выбор LLM и стека, дизайн API-контракта |
| 3-4 | Формирование Knowledge Base: индексация ключевых кодексов РК (Трудовой, Гражданский, Семейный) |
| 5-6 | Разработка Router Agent + 3 специализированных агента (трудовое, гражданское, семейное) |
| 7-8 | Разработка Response Composer + API-интеграция с amicus-service |
| 9 | UI модерации в amicus-crm |
| 10 | Тестирование на реальных вопросах (исторические данные из amicus), калибровка |

**Объём Phase 1:**
- 3 области права (трудовое, гражданское, семейное)
- Только русский язык
- 100% модерация
- Формат: один вопрос — один ответ (без follow-up)

**Критерии успеха Phase 1:**
- Approve rate > 80% на первых 200 ответах
- Время модерации < 2 мин на ответ
- Escalation rate < 30%
- Нет критических юридических ошибок

### Phase 2: Расширение + частичная автоматизация (4-6 недель)

- Добавление 3-5 областей права (корпоративное, налоговое, жилищное, земельное, административное)
- Автоматическая отправка ответов с confidence >= 0.85 для simple-вопросов
- Follow-up вопросы (контекст диалога)
- Казахский язык (P1)
- Расширение Knowledge Base: подзаконные акты, нормативные постановления ВС
- A/B-тестирование промптов

**Критерии перехода на Phase 2:**
- Phase 1 работает 4+ недели
- Approve rate стабильно > 85%
- 500+ обработанных ответов

### Phase 3: Автоматизация (ongoing)

- Автоматическая отправка большинства ответов (модерация только для низкого confidence и сложных)
- Кэширование типовых вопросов → мгновенные ответы
- AI-суммаризация для эскалированных вопросов
- Рекомендации платных услуг D24
- Spot-check модерация (5-10%)
- Предиктивная аналитика: какие темы набирают популярность

**Критерии перехода на Phase 3:**
- Phase 2 работает 8+ недель
- Approve rate стабильно > 95%
- 2000+ обработанных ответов
- 0 критических ошибок за 30 дней
