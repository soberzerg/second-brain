# PRD: Автоматизация колл-центра Базис

> Коммерческое предложение Clario для девелоперской компании «Базис»

## 1. Обзор проекта

### Проблема

Колл-центр Базис работает на ручном труде операторов с системными ограничениями:

- **Медленная обработка лидов:** 2-3 дня до первого контакта — лиды остывают и уходят к конкурентам
- **Ограниченный режим:** операторы работают до 18:00, а заявки поступают 24/7 — ночные и выходные лиды теряются
- **Человеческий фактор:** операторы не следуют скриптам, путают бренды (упоминают «Базис» при звонках по Нурлы Дала 2), импровизируют под стрессом
- **Нет мессенджеров:** отсутствуют бизнес-аккаунты WhatsApp и Telegram — менеджеры пишут с личных номеров без контроля
- **Нет сквозной аналитики:** невозможно измерить эффективность и оптимизировать процесс

### Решение

Комплексная платформа автоматизации на базе Clario + AGIents:

- **ИИ-голосовой ассистент** для исходящих звонков из CRM
- **Чат-боты** в WhatsApp и Telegram для дожима через мессенджеры
- **Автоматический flow:** звонок → недозвон → повторный звонок → мессенджер
- **Интеграция с CRM** Базиса через открытые API
- **Аналитика и контроль** качества коммуникаций

### Целевой результат

Сократить время обработки лида с 2-3 дней до **< 5 минут**, обеспечить покрытие **24/7**, повысить конверсию в квалифицированный лид (SQL).

## 5. Функциональные требования

### 5.1 Голосовой ИИ-ассистент

**Исходящие звонки:**
- Автоматический обзвон новых лидов из CRM (приоритет — свежие заявки)
- Адаптивный диалог на основе данных CRM: какой ЖК интересен, бюджет, ипотечные потребности
- Разделение скриптов по проектам:
  - Премиум-ЖК (Неопарк, Жандостар и др.) — скрипт с упоминанием «Базис»
  - Нурлы Дала 2 — отдельный скрипт, **без упоминания «Базис»**
- Сбор информации: предпочтения по планировкам, бюджет, тип оплаты (ипотека УРЫС, партнёрская, рассрочка)
- Цель диалога: запись на онлайн-консультацию или встречу в офисе продаж

**Обработка недозвонов:**
- До 3 попыток дозвона с настраиваемыми интервалами
- После неуспешных попыток — автоматический переход в мессенджер

**Передача оператору (Phase 1 — гибрид):**
- Детекция сложных/нестандартных ситуаций
- Живая передача (warm transfer) на оператора с контекстом диалога
- Оператор видит полную историю и данные из CRM

### 5.2 Чат-боты WhatsApp + Telegram

- Автоматическая отправка сообщения после неуспешного дозвона
- Адаптивный диалог (не шаблонное меню): ИИ ведёт беседу на основе данных CRM
- Отправка информации: планировки, цены, ипотечные условия, локация офиса
- Запись на консультацию / встречу через бота
- Поддержка мультимедиа: фото планировок, PDF-презентации ЖК
- Бизнес-аккаунты WhatsApp и Telegram (не личные номера)

### 5.3 Автоматический flow

```
Новый лид в CRM
       │
       ▼
  ┌─ Звонок (попытка 1) ──────── Успех → Квалификация → CRM
  │         │
  │    Недозвон
  │         │
  │    ⏳ Интервал (настраиваемый)
  │         │
  ├─ Звонок (попытка 2) ──────── Успех → Квалификация → CRM
  │         │
  │    Недозвон
  │         │
  │    ⏳ Интервал
  │         │
  ├─ Звонок (попытка 3) ──────── Успех → Квалификация → CRM
  │         │
  │    Недозвон
  │         │
  ▼
  Мессенджер (WhatsApp/Telegram)
       │
       ├── Клиент ответил → Диалог → Квалификация → CRM
       │
       └── Нет ответа 48ч → Отметка в CRM, повтор через N дней
```

### 5.4 CRM-интеграция

- Получение новых лидов через API (real-time или polling)
- Обогащение карточки лида результатами диалога:
  - Статус квалификации
  - Предпочтения (ЖК, планировка, бюджет, тип оплаты)
  - Запись на встречу (дата, время, формат)
  - Транскрипт разговора
- Передача квалифицированных лидов (SQL) в отдел продаж
- Синхронизация статусов (не обрабатывать лиды, уже взятые менеджером)

### 5.5 Аналитика и дашборды

- **Операционные метрики:** количество звонков, дозвонов, сообщений, конверсии по этапам
- **Качество:** соответствие скриптам (100% автоматически), sentiment-анализ
- **Эффективность:** время обработки лида, стоимость контакта, SQL rate
- **Сравнение:** бот vs оператор (конверсия, стоимость, скорость)
- **По городам и ЖК:** разбивка метрик по проектам и географии

## 6. Этапы внедрения

### Phase 1: Пилот — гибридная модель (8-10 недель)

**Объём:** 1 ЖК (рекомендация — Нурлы Дала 2, как наиболее стандартизированный)

| Неделя | Задача |
|--------|--------|
| 1-2 | Подключение к CRM Базиса (API-интеграция), настройка бизнес-WhatsApp |
| 3-4 | Настройка чат-ботов WhatsApp + Telegram для выбранного ЖК |
| 5-6 | Запуск голосового ИИ-ассистента (платформенное решение) |
| 7-8 | Настройка автоматического flow (звонок → мессенджер) |
| 9-10 | Тестирование, калибровка скриптов, обучение команды |

**Критерии успеха пилота:**
- Время обработки лида < 15 минут
- Конверсия в SQL не ниже текущей (baseline)
- Стоимость контакта ниже текущей на 30%+
- Положительная обратная связь от отдела продаж

**Модель работы Phase 1:**
- Бот делает первичный обзвон и квалификацию
- Сложные случаи и «горячие» лиды — передача живому оператору
- Операторы работают с подготовленным контекстом от бота
- Полный мониторинг и ручная корректировка скриптов

### Phase 2: Масштабирование (после успешного пилота)

- Подключение остальных ЖК (премиум-скрипты)
- Расширение на все города (Алматы, Астана, Шымкент, Атырау)
- Оптимизация голосового движка на основе данных пилота
- Расширенная аналитика и A/B-тестирование скриптов

### Phase 3: Полная автоматизация (6-12 месяцев)

- Бот ведёт весь диалог до записи на встречу без участия оператора
- Собственный voice pipeline (если экономически целесообразно)
- Предиктивная аналитика: оптимальное время звонка, канал, скрипт
- On-premise вариант развёртывания (если требование подтвердится)

## 8. ROI и бизнес-эффект

### Источники экономии

| Фактор | Эффект |
|--------|--------|
| Сокращение ручного труда | Бот обрабатывает 80%+ первичных контактов |
| Покрытие 24/7 | +30-40% обработанных лидов (ночные, выходные) |
| Скорость обработки | Лид получает контакт за минуты, а не дни — выше конверсия |
| Качество скриптов | 100% следование, нет путаницы брендов |
| Масштабируемость | Нет ограничения по количеству одновременных контактов |

### Ожидаемые результаты (горизонт 12 месяцев)

- Снижение стоимости контакта на **40-60%**
- Рост конверсии лид → SQL на **20-30%** за счёт скорости и покрытия
- ROI > 200% к концу первого года при полном масштабировании

## 9. Риски и митигация

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Качество голосового бота не устроит клиентов | Средняя | Высокое | Гибридная модель Phase 1, A/B-тесты бот vs оператор |
| Сложная интеграция с самописной CRM | Средняя | Среднее | Ранний старт интеграции, выделенный ресурс на стороне Базиса |
| Сопротивление внутри компании | Высокая | Среднее | Поэтапное внедрение, демонстрация метрик, обучение |
| Регуляторные ограничения (хранение данных, СПАМ-звонки) | Низкая | Высокое | Юридическая проверка, opt-in от клиентов, on-premise опция |
| Конкуренция за тендер — клиент выберет другого подрядчика | Средняя | Высокое | Быстрый запуск пилота, демонстрация готовых компонентов |
