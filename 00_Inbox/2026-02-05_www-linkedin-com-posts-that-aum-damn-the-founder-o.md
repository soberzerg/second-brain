# Создатель Claude Code раскрыл свой workflow

## Основная информация

**Boris Cherny** — создатель и лидер Claude Code в Anthropic. Присоединился к компании в сентябре 2024, начал с простого прототипа в терминале.

## Ключевые факты о разработке

### Скорость разработки (впечатляет!)
- **60-100 внутренних релизов в день**
- **~5 pull requests на инженера в день**
- Почти ежедневные публичные релизы
- При разработке todo-листа создал **20+ прототипов за 2 дня**

### Технологический стек
- **TypeScript + React** (UI)
- **Ink** — React-фреймворк для терминала
- **Yoga** — layout система для адаптивного размера терминала
- **Bun** — быстрая сборка и упаковка
- **npm** — дистрибуция

**Главный факт:** ~90% кода Claude Code написано с помощью Claude Code

### Философия разработки

#### "Product Overhang"
Прорыв случился когда добавили доступ к файловой системе — модель смогла самостоятельно исследовать кодовые базы. Возможности модели превосходили дизайн продукта.

#### Минимум кода вокруг модели
**Ключевой принцип:** минимизировать бизнес-логику, позволяя Claude обрабатывать сложность.

**Радикальный пример:** при выходе Claude 4.0 удалили ~50% системного промпта. Вместо наращивания scaffolding'а — упрощение.

#### Быстрая итерация > Планирование
Вместо длительного планирования — быстрые прототипы и итерации на основе фидбека.

### Архитектура
- Работает локально без виртуализации
- Система разрешений запрашивает подтверждение перед потенциально необратимыми действиями

## История создания

**Начало:** Простой прототип для отображения музыки через AppleScript
**Прорыв:** Добавление файлового доступа
**Результат:** Инструмент для разработки, который разрабатывает сам себя

## Инсайты для применения

### Что можно перенять:
1. **Радикальная простота** — удаляй код, не добавляй
2. **Быстрые итерации** — 20 прототипов лучше одного идеального плана
3. **Минимум scaffolding** — пусть ИИ делает сложное
4. **Dogfooding** — используй свой продукт для его создания
5. **Высокая скорость релизов** — не бойся частых деплоев

### Что НЕ работает:
- Длительное планирование перед действием
- Наращивание бизнес-логики вместо упрощения
- Боязнь удалять код

## Связь с моими проектами

- **AGIents.pro** — применить философию "минимум кода вокруг модели"
- **TG Claude Code Assistant** — использует ту же идею (CLI в --print mode)
- **Методика разработки** — брать пример с 5 PR/день на инженера
