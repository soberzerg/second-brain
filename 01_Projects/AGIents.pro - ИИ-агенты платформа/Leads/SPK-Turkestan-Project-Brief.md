## Заказчик

- **Компания:** АО «СПК «Түркістан» (Социально-предпринимательская корпорация Туркестанской области)
- **Тип:** Квазигосударственная корпорация при Акимате
- **Масштаб:** Крупная — проекты на миллиарды тенге (строительство, инвестиции, соц. инфраструктура)
- **Сайт:** spkturkistan.kz
- **Контакт через:** Аргын (знакомый Куаныша, руководящая должность в Акимате)

## Что хочет заказчик

### 1. Закрытый корпоративный ИИ-ассистент (основное)
- Семантический поиск по внутренним документам (уставы, регламенты, приказы, протоколы, проектная документация)
- Ответы с указанием источников (документ, дата, раздел/страница)
- Учёт редакций документов, сравнение версий
- Сводки и выжимки из длинных документов
- Помощь в подготовке служебных записок, справок на основе шаблонов
- RBAC — разграничение доступа по ролям (руководство, юристы, сотрудники, админ)
- Контролируемый выход во внешнюю сеть (НПА РК, гос. порталы) по whitelist

### 2. Внешний support-бот (дополнительно)
- Чат-бот для внешних пользователей (на сайте)
- Доступ для любого посетителя
- **Примечание:** Этого нет в КП конкурента — наше конкурентное преимущество

### 3. Формат подачи
- КП (коммерческое предложение) — готовит Сергей
- Презентация — для руководства (госструктура, привыкли к визуальным материалам)
- Язык: бизнес-язык, не слишком технический (КП конкурента критиковали за техничность)

## Инфраструктура (ключевые вопросы)

### Требования к контуру
- Квазигос → скорее всего, нужен on-premise / закрытый контур
- Данные не должны покидать пределы страны
- Нужно выяснить у заказчика: есть ли своя инфраструктура, где хостятся

### Варианты (предложить в КП)
1. **Своя инфра заказчика** — сервер с GPU ставится в их дата-центр
2. **Аренда GPU** — у казахстанского хостера + VPN до заказчика
3. **Гибридный** — веб-сервер отдельно, GPU-сервер отдельно

### Модели (архитектурное решение)
- Основная сильная модель (Gemini / Claude API или open-source для контура)
- Опционально: казахстанская ALM-LLM для перевода на казахский (для галочки + реальной пользы)
- Возможен «зоопарк» моделей: сильная + слабая + мини для простых задач
- Учесть: open-source модели быстро устаревают, fine-tuning дорогой и неэффективный

### Рекомендуемый сервер (из КП конкурента)
- CPU: 16 ядер / 32 потока (Xeon / EPYC)
- RAM: 128 GB ECC
- Диски: 2x1.92 TB NVMe (RAID1)
- Сеть: 1 Gbps
