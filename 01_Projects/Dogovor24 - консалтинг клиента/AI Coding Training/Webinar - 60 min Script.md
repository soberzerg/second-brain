### Слайд 6: Три мифа

## БЛОК 3: ОСНОВНОЙ КОНТЕНТ / ФРЕЙМВОРК [15:00 — 30:00]

### Слайд 8: Эволюция: три волны

**Визуал:** Timeline: 2021 (Autocomplete) → 2023-2025 (Chat & Vibe Coding) → 2025-2026 (Agentic Engineering)

**Текст спикера:**

> "Три волны.
>
> **2021** — Autocomplete. Copilot. ИИ подсказывает строку. Как автозамена в телефоне.
>
> **2023-2025** — Chat и Vibe Coding. Описываешь фичу, ИИ генерирует блок. Copy-paste. Быстро, весело, хрупко. Karpathy сам сказал: 'It was good fun and it almost worked.'
>
> **2025-2026** — Agentic Engineering. Вы говорите: 'Нужна авторизация через Google'. Агент планирует, пишет, тестирует, находит баг, чинит, тестирует снова.
>
> Вы больше не пишете код. Вы — менеджер ИИ-команды."

### Слайд 9: Фреймворк четырёх уровней (пирамида)

**Визуал:** Пирамида снизу вверх: Tutor → Assistant → Agent Manager → AI System Architect

> **Уровень 1: Tutor — ИИ как учитель.**
>
> Когда: изучаете новый фреймворк, дебажите незнакомый код, разбираетесь в legacy.
>
> Ключевое правило: **выключите автокомплит!** Если ИИ дописывает за вас — вы не учитесь. Chat mode. Просите ИИ задавать вопросы, а не давать ответы. Файл tutor.md: "ты мой учитель, веди через вопросы".
>
> **Уровень 2: Assistant — ИИ как помощник.**
>
> Когда: знакомая область, знаете что делать, не хотите писать boilerplate.
>
> Теперь автокомплит ON. Генерируете схемы, тесты, валидацию. Но понимаете каждую строчку.
>
> Правило: если не понимаете, что ИИ сгенерировал — возвращайтесь на Уровень 1. Это не стыдно. Это инженерная дисциплина.

### Слайд 10: Уровни 3 и 4 (ключевые)

> **Уровень 3: Agent Manager — вы менеджер ИИ-команды.**
>
> Здесь магия. Вы не просите "напиши функцию". Вы говорите "спроектируй систему уведомлений".
>
> Но прежде чем агент начнёт кодить — **spec-first**. Я называю это **"Waterfall за 15 минут"**:
> - Описываете задачу
> - ИИ задаёт уточняющие вопросы (типы уведомлений? синхронно или асинхронно? rate limiting?)
> - Вы отвечаете → ИИ компилирует в spec.md
> - Только после этого — план, только после плана — код
>
> Снижает переделки на **40-60%**. Это данные Addy Osmani, Head of Google Chrome Developer Experience.
>
> Дальше: **test-first**. ИИ генерирует тесты ДО кода. Потом реализация. Тесты проходят — коммитим. Покрытие растёт на **30-40%**.
>
> И **security review** на каждый PR. +40% уязвимостей = ревью не опция, а необходимость.
>
> **Уровень 4: AI System Architect — вы строите систему для всей команды.**
>
> CLAUDE.md с инструкциями для агента. Навыки агентов — плагины, субагенты. AI-код-ревью на каждом PR как часть CI/CD. Процесс улучшения на основе feedback loop.

### Слайд 11: Сдвиг навыков

**Визуал:** Таблица "Было → Стало"

| Было | Стало |
|------|-------|
| Prompt Engineering | Solution Architecture |
| Написание кода | Code Review at Scale |
| Отладка | Дизайн Feedback Loop |
| Тестирование | Continuous Validation |

> **Золотое правило:** *"Never commit code you can't explain."* Если ИИ сгенерировал что-то, что вы не понимаете — вы НЕ коммитите. Точка.

### Слайд 12: Actionable takeaway — "Waterfall за 15 минут"

**Визуал:** Пошаговая схема: Идея → Brainstorm с ИИ (5 мин) → spec.md (5 мин) → Project plan (5 мин) → Итеративный кодинг

> "Хотите попробовать прямо после вебинара?
>
> **Шаг 1:** Берёте задачу из бэклога. Одну фичу.
>
> **Шаг 2:** Чат с ИИ. Описываете идею. НЕ просите писать код! 'Хочу добавить систему уведомлений. Задай мне 10 уточняющих вопросов.' 5 минут — проработанная спецификация.
>
> **Шаг 3:** ИИ компилирует в spec.md: requirements, архитектура, data models, стратегия тестирования, edge cases.
>
> **Шаг 4:** Разбить на задачи. По каждому шагу — сначала тесты, потом реализация.
>
> **Шаг 5:** Реализация по chunk-ам. Каждый chunk: тест → код → ревью → коммит. Не 500 строк. 30-50 строк.
>
> 15 минут на планирование экономят часы на переделках."

**Интерактив (мин. 20):** "На каком вы уровне? Напишите в чат 1, 2, 3 или 4"

## БЛОК 4: КЕЙСЫ И ДОКАЗАТЕЛЬСТВА [30:00 — 45:00]

### Слайд 13: Кейс 1 — Security Audit

**Визуал:** Блок кода с подсветкой проблемы

```php
class UserController {
    public function search(Request $request) {
        $query = $request->get('q');
        $users = DB::select(
            "SELECT * FROM users WHERE name LIKE '%{$query}%'"
        );
        return response()->json($users);
    }
}
```

> "Кто видит проблему? Классическая SQL-инъекция. $query вставляется напрямую. ИИ сгенерировал уверенно. Код выглядит чисто. Работает на тестовых данных. Бомба замедленного действия на проде.
>
> За 6 аудитов мы нашли: SQL-инъекции, XSS, захардкоженные секреты, проблемы с авторизацией. Каждый раз.
>
> Security-first review — это не перестраховка. Это гигиена."

### Слайд 14: Кейс 2 — Обучение команды

**Визуал:** "До" и "После" в виде двух колонок

> "До курса: разработчики использовали ИИ как 'умный автокомплит'. Copy-paste из ChatGPT. Тест-покрытие низкое. Spec-first? Никто не слышал.
>
> Ожидаемые результаты через 3 месяца практики:
> - −25-40% на рутинных задачах
> - +30-50% к тест-покрытию
> - −40-60% уязвимостей
> - −20-30% багов в продакшене
>
> Самая частая фраза после обучения: 'Теперь я понимаю, почему код ломался — я просто не давал ИИ достаточно контекста.'"

### Слайд 15: Кейс 3 — Личная продуктивность

**Визуал:** Мокап Telegram-бота

>
> Написал за один вечер. Я описал архитектуру, дал контекст, разбил на chunk-и — агент сделал реализацию. Я ревьюил, тестировал, направлял. Spec-first даже для pet-проекта.

> Vibe coding? Нет. Agentic engineering за один вечер. Разница — мне не пришлось два дня фиксить баги."

**Визуал:** Разбивка экрана — слева код, справа spec.md
