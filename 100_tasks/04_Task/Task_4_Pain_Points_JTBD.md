## Часть 2: Jobs to Be Done (JTBD)

### Методология JTBD

> "Люди не покупают продукты, они нанимают их для выполнения работы."
> — Clayton Christensen

**Формат:** When [ситуация], I want to [мотивация], so I can [ожидаемый результат].

---

### 2.1 Functional Jobs (Функциональные задачи)

#### Job #1: Автоматизировать ответы на типовые вопросы

```
When: клиенты задают одни и те же вопросы 24/7
I want to: автоматически отвечать на них без участия человека
So I can: освободить сотрудников для сложных задач и не терять клиентов ночью
```

**Метрики успеха:**
- % автоматически обработанных запросов
- Время ответа (секунды vs минуты/часы)
- Удовлетворённость клиентов

---

#### Job #2: Быстро запустить ИИ-агента без разработчиков

```
When: я хочу ИИ-агента, но у меня нет технической команды
I want to: создать агента самостоятельно без написания кода
So I can: запуститься за дни, а не месяцы, и не тратить бюджет на разработку
```

**Метрики успеха:**
- Время от регистрации до работающего агента
- Количество шагов настройки
- Нужна ли помощь разработчика (да/нет)

---

#### Job #3: Использовать свою базу знаний в агенте

```
When: у меня есть информация о компании (услуги, цены, FAQ)
I want to: легко загрузить её в агента
So I can: агент отвечает точно и актуально, а не выдумывает
```

**Метрики успеха:**
- Время загрузки базы знаний
- Точность ответов агента
- Простота обновления информации

---

#### Job #4: Масштабировать решение на несколько объектов

```
When: у меня сеть (санаториев, магазинов, офисов)
I want to: быстро раскатить агента на новые точки
So I can: не тратить 2-3 недели на каждый объект
```

**Метрики успеха:**
- Время добавления нового объекта
- % переиспользуемых настроек
- Централизованное управление

---

### 2.2 Emotional Jobs (Эмоциональные задачи)

#### Job #5: Чувствовать уверенность в технологии

```
When: я слышу про ИИ и автоматизацию
I want to: понимать, что это работает и не сломается
So I can: принять решение без страха "выбросить деньги"
```

**Как решить:**
- Case studies из своей отрасли
- Trial период
- Поддержка на русском

---

#### Job #6: Выглядеть инновационным перед руководством

```
When: я предлагаю внедрить ИИ-агента
I want to: показать конкретный ROI и успешные примеры
So I can: получить одобрение бюджета и выглядеть профессионалом
```

**Как решить:**
- ROI калькулятор
- Презентационные материалы
- Метрики для отчётов

---

### 2.3 Social Jobs (Социальные задачи)

#### Job #7: Не отстать от конкурентов

```
When: я вижу, что конкуренты внедряют ИИ
I want to: тоже иметь ИИ-решение
So I can: не потерять клиентов и не выглядеть отсталым
```

**Как решить:**
- Быстрый запуск (опередить конкурентов)
- "Наши клиенты уже используют" messaging

## Часть 3: User Stories для MVP

### Формат

> As a [роль], I want [действие], so that [результат].

---

### Epic 1: Быстрый старт

```
US-1: As a business owner, I want to create an AI agent from a template,
      so that I can launch in hours, not weeks.

US-2: As a non-technical user, I want to configure the agent without coding,
      so that I don't need to hire developers.

US-3: As a new user, I want to see a demo before signing up,
      so that I understand what I'm buying.
```

---

### Epic 2: База знаний

```
US-4: As a business owner, I want to upload my documents (PDF, Word, Excel),
      so that the agent knows about my business.

US-5: As an admin, I want to add FAQ manually,
      so that I can ensure correct answers to common questions.

US-6: As an admin, I want to update prices/information easily,
      so that the agent always has current data.
```

---

### Epic 3: Работа агента

```
US-7: As a customer, I want to get instant answers 24/7,
      so that I don't have to wait for business hours.

US-8: As a customer, I want to be transferred to human when needed,
      so that complex issues are handled properly.

US-9: As a business owner, I want the agent to book appointments/reservations,
      so that I capture leads automatically.
```

---

### Epic 4: Аналитика

```
US-10: As an admin, I want to see how many conversations the agent handled,
       so that I can measure ROI.

US-11: As an admin, I want to see what questions the agent couldn't answer,
       so that I can improve the knowledge base.
```

---

### Epic 5: Self-hosted (RU market)

```
US-12: As an enterprise customer, I want to deploy on my servers,
       so that customer data stays within my infrastructure.

US-13: As an IT admin, I want clear deployment documentation,
       so that I can set up without external help.
```
