# Выжимка: The ULTIMATE Claude Code Tutorial - 27 Concepts in 27 Minutes

**Источник:** https://youtu.be/ZlDnsf_DOzg?si=hYOW3nCc7F9KubOF
**Дата:** 2026-03-02

## Основная тема

Подробное руководство по Claude Code — инструменту, который в отличие от чат-ботов может выполнять реальные действия на компьютере (создавать файлы, строить сайты, настраивать БД, автоматизировать задачи). Автор объясняет 27 ключевых концепций, ориентируясь прежде всего на нетехнических фаундеров.

## Ключевые тезисы

1. **Claude Code — action-oriented AI, а не чат-бот** - В отличие от ChatGPT, Claude или Gemini, Claude Code способен выполнять действия на компьютере: создавать файлы, строить веб-сайты, настраивать базы данных, устанавливать пакеты и автоматизировать задачи.

2. **Terminal не требует глубоких знаний CLI** - Вместо сложных команд вы вводите запросы на простом английском. Claude сам преобразует их в команды. Базовые команды: `claude` (запуск), `Ctrl+C` (двойное — выход), `/clear` (очистка).

3. **Качество зависит от точности промптов** - Вместо "Создай мне веб-сайт" лучше "Создай одностраничный лендинг для консалтинга с синей схемой, тремя карточками услуг и контактной формой". Чем конкретнее промпт — тем точнее результат.

4. **Permissions — критически важная система** - Claude Code может создавать, запускать, редактировать и удалять файлы. Безопасные действия можно предварительно одобрить в `settings.json`, но рискованные операции (установка пакетов, удаление файлов, доступ к сети) должны требовать подтверждения. Флаг `--dangerously-skip-permissions` использовать не стоит.

5. **Tools выбираются автоматически** - У Claude Code есть встроенные инструменты: `read` (чтение файлов), `write` (создание), `bash` (команды). Достаточно описать задачу — Claude сам определит какие инструменты использовать.

6. **Context Window = кратковременная память с ограничениями** - При перегрузке информацией Claude начинает "забывать" ранние детали (Context Rot). Решение: фокусировать беседы или использовать `/clear` и `/compact`.

7. **Conversation History сохраняется между сессиями** - Каждая сессия автоматически сохраняется. Команда `claude --resume` позволяет продолжить работу с полным контекстом предыдущего взаимодействия.

8. **Token Usage определяет стоимость** - ~1 токен = 0.75 слова. Sonnet дешевле и универсальнее, Opus мощнее для сложных задач но дороже. Отслеживание: `/cost` и `/stats`.

9. **CLAUDE.md — главный инструмент персонализации** - Markdown-файл с предпочтениями, правилами и структурой проекта. Claude читает его при каждом запуске. Избавляет от "догадок" AI и гарантирует последовательные результаты.

10. **Два типа долгосрочной памяти** - Auto memory (автоматически сохраняет контекст, паттерны, предпочтения) и `CLAUDE.md` (заданные вами инструкции). Обе загружаются в контекст при каждой новой сессии.

## Полезные инсайты

- **Context Rot — реальная проблема** - AI может "забывать" раннюю информацию в длинных разговорах. Это подчёркивает важность структурирования бесед и использования `/compact` для сжатия контекста.

- **CLAUDE.md как "обучение" AI** - Файл позволяет сделать поведение Claude Code предсказуемым и персонализированным в каждом проекте, заменяя бесконечные повторения одних и тех же инструкций.

- **Sub-agents с изолированным контекстом** - Делегирование задач специализированным суб-агентам значительно улучшает точность: каждый работает в чистом контексте, без "шума" основной беседы.

- **Визуальная коммуникация через скриншоты** - Вставка изображений (ошибка на сайте, макет дизайна) быстрее и точнее текстового описания. Claude может "прочитать" изображение и создать код.

- **Worktrees для параллельной разработки** - Изолированные рабочие деревья позволяют запускать несколько экземпляров Claude Code над разными задачами в одном проекте одновременно.

- **Agent Teams — прямое взаимодействие агентов** - Новая функция позволяет нескольким агентам (коллегам) напрямую общаться друг с другом и совместно использовать список задач, минуя главного агента.

- **Баланс безопасности и скорости** - Система разрешений даёт полный контроль, позволяя найти оптимальный баланс между ручным подтверждением и автоматическим выполнением.

## Практические рекомендации

- [ ] **Создать `CLAUDE.md` для каждого проекта** - Определить стиль кодирования, правила, предпочтения и структуру
- [ ] **Настроить permissions в `settings.json`** - Предварительно одобрить безопасные действия (git status, чтение файлов), требовать подтверждения для рискованных
- [ ] **Использовать `/compact` при длинных сессиях** - Указать что сохранить, чтобы избежать Context Rot
- [ ] **Выбирать модель под задачу** - Sonnet для повседневных задач, Opus для сложных проблем, Haiku для быстрых простых операций
- [ ] **Создать slash commands для рутинных задач** - Поместить в папку `.claude/commands` для автоматизации
- [ ] **Использовать Worktrees для параллельной работы** - Флаг `-w` для изолированных веток при одновременной разработке нескольких фич
- [ ] **Подключить MCP-серверы** - Расширить возможности Claude Code интеграциями с Airtable, Notion, Slack, Gmail и другими
