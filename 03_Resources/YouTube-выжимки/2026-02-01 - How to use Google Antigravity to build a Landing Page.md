# Выжимка: How to use Google Antigravity to build a REAL Landing Page for a LEAD MAGNET

**Источник:** https://www.youtube.com/watch?v=YqO9LA09Mig

## Основная тема

Демонстрация создания полноценной конверсионной целевой страницы для лид-магнита с использованием Google Antigravity — AI-платформы для кодирования. Показана интеграция с базой данных Supabase и развертывание на Vercel. Фокус на том, как Antigravity использует подробные инструкции и "skills" для автоматизации всего цикла разработки.

## Ключевые тезисы

1. **Antigravity как "конструктор Lego"** — Каждый компонент (инструкция, "skill", ресурс) является простым "кирпичиком", который объединяется в сложную, но работающую систему. Это позволяет создавать сложные решения из предсказуемых, модульных частей.

2. **Трехуровневая архитектура агента:**
   - **Directive (what to do):** Инструкции для системы
   - **Orchestration (decision-making):** Агент интерпретирует директивы, обрабатывает ошибки, управляет направлениями
   - **Execution (doing the machine):** Детерминированные Python-скрипты, вызовы API, операции с файлами

3. **"Skills" как мощный инструмент для ИИ** — Заранее определенные наборы инструкций, позволяющие Antigravity глубоко понимать конкретные задачи. Например, `SKILL_FrontendDesign.md` содержит детальные указания по созданию UI. Избегается "раскручивание колес" (трата токенов на догадки).

4. **Создание бренд-гайдов из референсов с помощью ИИ** — Референсные изображения с Behance → ChatGPT 4.0 для извлечения текстовых описаний → файл `Brand.md` → Antigravity генерирует новые Brand Guidelines с шрифтами, палитрами и стилями.

5. **Автоматическое создание Implementation Plans** — Antigravity генерирует подробные планы, описывающие шаги реализации. Пользователь может просматривать и корректировать поведение ИИ через комментарии.

6. **Интеграция через MCP (Model Context Protocol)** — Бесшовное подключение к GitHub (контроль версий) и Supabase (база данных лидов — имена и email).

7. **Развертывание на Vercel** — После создания кода и сохранения на GitHub, проект быстро публикуется через Vercel.

## Полезные инсайты

- ИИ можно использовать как "арт-директора" — давать обратную связь и указания по визуальному дизайну
- Текстовые описания референсов (почему дизайн сделан именно так) значительно улучшают понимание ИИ
- Skills можно легко переносить между проектами для переиспользования проверенных инструкций
- Архитектура "Directive-Orchestration-Execution" обеспечивает предсказуемость работы ИИ vs "черный ящик" других инструментов

## Практические рекомендации

- [ ] Собирать подробные референсы — не только визуальные, но и текстовые описания (бренд-гайды, концепции дизайна)
- [ ] Использовать ChatGPT для извлечения текста из изображений/документов
- [ ] Создавать собственные "Skills" для повторяющихся задач и стандартизации
- [ ] Применять 8-шаговую архитектуру для лендингов: Hero → Success → Problem-Agitate и т.д.
- [ ] Регулярно просматривать и корректировать планы реализации ИИ
- [ ] Подготовить собственные активы (стикеры, фоны) для использования ИИ
- [ ] Интегрировать проект со специализированными сервисами (GitHub, Supabase, Vercel)
