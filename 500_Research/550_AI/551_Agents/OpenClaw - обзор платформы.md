# OpenClaw — обзор платформы

## Что такое OpenClaw

OpenClaw — open-source фреймворк для создания персональных автономных ИИ-ассистентов. Превращает LLM (Claude, GPT, Gemini или любую open-source модель с совместимым API) из реактивного чатбота в автономного оператора, способного выполнять реальные задачи на компьютере пользователя.

Создан австрийским разработчиком **Peter Steinberger** в ноябре 2025 года под именем Clawdbot, переименован в OpenClaw в январе 2026. За ~60 дней набрал 250,000+ звёзд на GitHub — один из самых быстрорастущих open-source проектов в истории.

Проект независим от Anthropic, хотя изначально построен на Claude.

## Ключевая концепция: Agent Harness

OpenClaw — это **agent harness** (оснастка агента): фреймворк, который оборачивает LLM дополнительными слоями (память, инструменты, триггеры, инструкции, каналы вывода), превращая его из one-turn ассистента в автономного агента.

### Пятислойная архитектура

1. **Модель (мозг)** — LLM как движок рассуждений. Без инфраструктуры stateless.
2. **Gateway & Session Layer** — всегда-включённый сервис: маршрутизация сообщений, управление сессиями, персистенция истории разговоров.
3. **Context Management & Memory** — два типа памяти:
   - **Working memory** — текущие разговоры и состояние сессии
   - **Retrieved memory** — поисковая система для извлечения релевантного исторического контекста
   - Проблема: **context rot** — деградация производительности при накоплении файлов памяти, схем инструментов и саммари.
4. **Instructions, Tools & Agentic Loop** — системные промпты, схемы инструментов (браузер, файлы, код, интеграции, cron), и цикл обратной связи (act → observe → continue).
5. **Triggers & Outputs** — механизмы пробуждения (таймеры, cron, webhooks) и выход за пределы ответов (файлы, память, участие в рабочих процессах).

### Четыре ключевых вопроса дизайна

Понимание любого agent harness сводится к 4 вопросам:
1. Что инициирует активацию агента?
2. Какой контекст инжектируется каждый ход?
3. Какие инструменты доступны?
4. Какие выходные данные / изменения в мире возможны?

## Возможности

### Каналы связи
- WhatsApp, Telegram, Discord, Slack, Signal, iMessage
- Работает в личных сообщениях и групповых чатах

### Технические возможности
- **Управление браузером** — навигация, заполнение форм, веб-скрапинг
- **Полный доступ к системе** — файловые операции, shell-команды, выполнение скриптов
- **Персистентная память** — учится на предпочтениях и контексте пользователя
- **Система скиллов** — 100+ предустановленных плагинов + возможность создания кастомных
- **Самомодификация** — может писать и обновлять собственные скиллы
- **Автономные задачи** — cron-задачи, напоминания, фоновые процессы
- **24/7 работа** — работает локально на машине пользователя

### Практические сценарии
- Управление email: очистка инбокса, отправка писем, автоматическая отписка
- Управление календарём, подготовка к встречам, чек-ин на рейсы
- Code review и автономные циклы тестирования
- Мульти-агентная оркестрация между машинами
- Управление IoT-устройствами
- Персональный knowledge management
- Агрегация и анализ данных о здоровье
- Создание контента и постинг в соцсети

### Интеграции
50+ интеграций: Gmail, GitHub, Obsidian, Spotify, Twitter и множество productivity-инструментов.

## Установка

- **npm one-liner** — быстрая установка
- **Homebrew** — для macOS
- **Git clone** — для продвинутых пользователей (hackable install)
- **macOS menubar app** (beta) — companion-приложение

## Модель лицензирования

- **Open-source**, бесплатный
- **BYOK** (Bring Your Own Keys) — пользователь приносит свои API-ключи
- **Model-agnostic** — Claude, GPT, Gemini или open-source модели

## Бенчмарки

OpenClaw в паре с Sonnet 4.6 в настроенной конфигурации harness показывает **79.6%** — превышает raw model score. Это демонстрирует, что оркестрация на уровне harness (prompt engineering, выбор инструментов, retry policies) способна материально улучшить результаты.

## Безопасность

Партнёрство с **VirusTotal** для верификации безопасности скиллов — ответ на обеспокоенность по поводу выполнения произвольного кода.

## Ограничения

- Широкие agent harness несут overhead, неоптимальный для узких задач
- Специализированные "sniper agents" часто превосходят универсальные системы в production
- Проблема **context rot** при накоплении контекста
- Для enterprise рекомендуются: чистые промпты, ограниченные инструменты, минимальная память, жёсткие границы
