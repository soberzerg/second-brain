# Выжимка: Как я использую Claude Code для кодинга больших проектов

**Источник:** https://www.youtube.com/watch?v=PwtfglP241c
**Дата:** 2026-02-20

## Основная тема

Подробное практическое руководство по использованию Claude Code для разработки ПО — настройка, управление контекстом, субагенты, кастомные команды, интеграция с GitHub и MCP. Цель — показать, как эффективно применять AI в повседневной работе над крупными проектами.

## Ключевые тезисы

1. **Claude Code — платный инструмент без пробной версии** - Подписка от $17-20/мес. Использование через API "адски дорогое" — расходы быстро выходят из-под контроля. Автор успешно работает на минимальной подписке за $20/мес.

2. **Контекст — ключевой аспект работы** - Claude Code постоянно собирает контекст из проекта, его изменений и истории диалога. От объёма контекста (токенов) напрямую зависят затраты и качество ответов AI.

3. **CLAUDE.md — основная документация для AI** - Зарезервированный Markdown-файл, создаваемый командой `/init`. Содержит описание архитектуры, правила, команды. Claude использует его как первичный источник контекста при каждом запросе. При инициализации можно указать дополнительные файлы: `/init учитывай правила @docs/architecture-rules.md`.

4. **Управление контекстом критически важно для экономии токенов** - Переполнение контекста ведёт к автоматическому `auto-compact` в неподходящий момент. Рекомендуется вручную использовать `/compact` для суммирования истории или `/clear` для полной очистки при переключении между несвязанными задачами.

5. **Plan mode позволяет контролировать действия AI** - Включается через Shift+Tab. Claude сначала предлагает подробный план, задаёт уточняющие вопросы и только после одобрения приступает к выполнению. Особенно полезен для сложных задач, требующих продуманного архитектурного подхода.

6. **Субагенты специализируются на конкретных задачах** - Настраиваемые мини-агенты с собственным контекстом, системным промптом и набором инструментов. Позволяют делегировать узкоспециализированные функции (code review, проверка безопасности) без перегрузки основного контекста, повышая точность и снижая затраты.

7. **Кастомные команды автоматизируют рутину** - Markdown-файлы в `.claude/commands/` задают шаблоны команд (например, `code-review.md`), которые вызываются через `/` и принимают аргументы через `$ARGUMENTS`.

8. **Интеграция с GitHub через `gh` CLI** - Claude автоматически создаёт коммиты, пушит изменения и формирует Pull Requests с подробным описанием фичи, тестовым планом и другими деталями.

9. **MCP расширяет возможности Claude** - Model Context Protocol позволяет подключаться к внешним инструментам (Sentry, ClickUp, Figma, Puppeteer) и источникам данных, превращая Claude из чат-бота в оркестратор инструментов разработки.

## Полезные инсайты

- **"Чем меньше, тем лучше" для контекста** - Оптимальное управление объёмом контекста напрямую влияет на стоимость и скорость. Сохранение только релевантной информации позволяет Claude быстрее и точнее фокусироваться на задаче.

- **AI-driven разработка ≠ автокомплит** - Claude Code предназначен для делегирования задач агентам и работы с большими блоками кода, а не для посимвольного автодополнения, характерного для обычных IDE.

- **Инкрементальные коммиты AI-кода** - Регулярное создание коммитов после каждого удовлетворительного изменения позволяет эффективно отслеживать прогресс, легко откатываться и поддерживать качество кодовой базы.

- **AI как оркестратор инструментов** - MCP и кастомные команды превращают Claude Code в мощный инструмент оркестрации — не просто чат, а центр управления всей инфраструктурой разработки.

## Практические рекомендации

- [ ] **Мониторить расход токенов** - Регулярно проверять `/usage` и `/cost`, чтобы оставаться в рамках подписки
- [ ] **Использовать `/clear` при переключении задач** - Полная очистка контекста экономит токены и повышает качество ответов
- [ ] **Применять `/compact` при разрастании диалога** - Создаёт краткий обзор беседы, высвобождая токены
- [ ] **Выносить объёмные инструкции в отдельные `.md` файлы** - Не засорять основной контекст, ссылаться на файлы в промптах
- [ ] **Детально заполнять CLAUDE.md** - Архитектура, правила, структура модулей — всё что поможет Claude давать точные ответы
- [ ] **Включать Plan mode (Shift+Tab) для сложных задач** - Сначала план, потом выполнение
- [ ] **Создавать субагентов для узких задач** - Через `/agents` для code review, анализа безопасности и т.д.
- [ ] **Автоматизировать через кастомные команды** - `.claude/commands/*.md` для частых операций
- [ ] **Всегда ревьюить AI-код** - Человеческий контроль обязателен даже при корректных на вид изменениях
- [ ] **Интегрировать GitHub CLI (`gh`)** - Автоматизация PR, коммитов и взаимодействия с репозиторием

## Упомянутые ресурсы

- **Claude Code** (инструмент) - Основной продукт от Anthropic для AI-driven разработки
- **Neovim + AstroNvim** (редактор) - Используется автором как основной редактор
- **Visual Studio Code** (IDE) - Альтернативная среда разработки
- **GitHub CLI (`gh`)** (утилита) - Командная строка для работы с GitHub
- **PurpleSchool** (образовательная платформа) - Платформа Антона Ларичева, спонсор видео
- **MCP (Model Context Protocol)** (стандарт) - Протокол интеграции AI с внешними инструментами
- **Sentry** (инструмент) - Мониторинг ошибок, подключается через MCP
- **ClickUp / Jira / Asana / Linear** (инструменты) - Таск-трекеры, интегрируемые через MCP
- **Puppeteer** (инструмент) - Браузер-автоматизация через MCP
- **Hugging Face** (платформа) - AI/ML модели, подключается через MCP
