## Целевая аудитория

**Сегмент:** Фаундеры, соло-предприниматели, CTO — те, кто работает руками и хочет системность.

**Портрет:**
- 28-45 лет, техническое мышление (не обязательно разработчик)
- Ведёт 2-5 проектов одновременно, всё в голове или разбросано по Notion/заметкам/чатам
- Использует ChatGPT / Claude, но бессистемно — "по запросу"
- Чувствует: знаний и контекста много, но они теряются. Каждую сессию с ИИ начинает с нуля
- Мечтает: ИИ, который знает его бизнес, проекты, клиентов — и не нужно каждый раз объяснять заново

**Ключевая боль:**
"У меня в голове — 5 проектов, 10 клиентов, 100 идей. Я трачу больше времени на то, чтобы вспомнить контекст, чем на принятие решений. ИИ мог бы помочь — но он не знает моего бизнеса."

**Ключевое желание:**
"Хочу систему, где всё моё знание — проекты, клиенты, решения, идеи — лежит в одном месте, и ИИ может с этим работать."

### README.md (главный файл — продающий + инструкция)

```markdown
# Второй мозг фаундера

> Система управления знаниями для тех, кто ведёт несколько проектов
> и хочет, чтобы ИИ знал их бизнес.

## Проблема

Вы ведёте 2-5 проектов. Клиенты, задачи, идеи, решения —
всё в голове, в Notion, в заметках, в чатах.

Каждый раз, когда вы открываете ChatGPT или Claude,
вы начинаете с нуля: объясняете контекст, описываете проект,
напоминаете, что уже решили.

**Вы тратите больше времени на передачу контекста,
чем на принятие решений.**

## Решение

Этот шаблон — готовая система "Второй мозг":

- **Obsidian** — хранилище всех знаний (проекты, клиенты, идеи, ресурсы)
- **Claude Code** — ИИ-ассистент, который читает ваше хранилище и знает контекст
- **CLAUDE.md** — "мозг системы": файл, который рассказывает ИИ о вашем бизнесе
- **Git** — синхронизация и история всех изменений
- **Готовые команды** — /thinking-partner, /daily-review, /inbox-processor

## Что вы получаете

1. **ИИ, который знает ваш бизнес.** Не нужно каждый раз объяснять контекст.
   Открыли Claude Code в папке хранилища — он уже знает ваши проекты,
   клиентов, текущие задачи.

2. **Структура PARA.** Проверенная система организации знаний:
   Projects (активные проекты), Areas (зоны ответственности),
   Resources (полезные материалы), Archive (завершённое).

3. **Готовые команды.** Не нужно писать промпты с нуля.
   `/thinking-partner` — думай вместе со мной.
   `/daily-review` — подведи итоги дня.
   `/inbox-processor` — разбери входящие.

4. **Git = машина времени.** Каждое изменение сохранено.
   Можно откатиться, посмотреть историю, синхронизировать
   между устройствами.

## Быстрый старт (15 минут)

### 1. Скопируйте репозиторий

```bash
git clone https://github.com/aisobolev/founder-second-brain.git my-brain
cd my-brain
```

### 2. Откройте в Obsidian

File → Open Vault → выберите папку `my-brain`

### 3. Заполните CLAUDE.md

Откройте файл `CLAUDE.md` и заполните шаблон:
- Кто вы (имя, роли, компании)
- Ваши проекты (название, описание, стек)
- Ваши клиенты
- Как вы работаете (предпочтения, стиль)

**Это самый важный шаг.** Чем точнее вы заполните CLAUDE.md,
тем умнее будет ваш ИИ-ассистент.

### 4. Создайте первый проект

Скопируйте папку `01_Projects/_Template - Проект/`,
переименуйте и заполните README.

### 5. Запустите Claude Code

```bash
cd my-brain
claude
```

Попробуйте: "Расскажи, что ты знаешь о моих проектах"

**Если "ChatGPT / Claude в браузере":**

Понял. Тогда вот главное, что нужно знать:

ChatGPT и Claude в браузере — это **разговор без памяти**. Каждый новый чат начинается с нуля. Вы тратите 5-10 минут, чтобы объяснить контекст, прежде чем получить полезный ответ.

**Claude Code** работает иначе. Это CLI-инструмент, который запускается в папке вашего проекта. Он видит все файлы. А файл **CLAUDE.md** рассказывает ему о вашем бизнесе, проектах, клиентах.

Результат: ИИ, который помнит всё. Не потому что у него "память", а потому что всё записано в вашей системе.

Отлично, значит вы в точке, где всё начинается.

## Описание трипваера: Часовая консультация "Настройка под ваш бизнес"

- Как вы работаете сейчас (какие инструменты, где теряете время)

**Блок 2 (25 мин): Адаптация CLAUDE.md**
- Совместно пишем/переписываем CLAUDE.md под ваш бизнес
- Правильная структура: профиль → проекты → клиенты → процессы → стиль
- Показываю свой CLAUDE.md (12 000 символов) как пример — что работает и почему

**Блок 3 (15 мин): Workflow + кастомные команды**
- Показываю мой ежедневный workflow: утро → работа → ревью
- Создаём 2-3 кастомные команды под ваши задачи

**Блок 4 (5 мин): Дальнейшие шаги**
- Чек-лист первой недели с системой
- Как обновлять CLAUDE.md по мере изменений
