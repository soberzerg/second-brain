# Git для вайбкодера — мини-гайд без терминала

**Формат:** PDF / статья-гайд
**Связанный пост:** [[2026-04-22 Вайб-кодинг лайфхак 5 Git]]
**Аудитория:** вайбкодеры и предприниматели, которые кодят с ИИ-агентом, но не работали с гитом
**Обещание:** за 15 минут ты начнёшь коммитить, откатывать и ветвить проект через интерфейс VS Code. Без терминала.

---

## Зачем тебе это читать

Ты кодишь с агентом. Агент уверенно делает то, что ты просишь. Иногда слишком уверенно — удаляет нужное, переписывает рабочее, ломает три файла ради исправления одного.

Гит решает четыре проблемы:

1. **Не теряешь файлы.** Каждая "точка сохранения" — это слепок всего проекта.
2. **Откатываешь ошибки агента.** Одно нажатие — и проект возвращается к рабочей версии.
3. **Тестируешь гипотезы.** Три параллельные ветки = три подхода к одной задаче.
4. **Хранишь проект в облаке.** GitHub = бэкап и доступ с любого компьютера.

Весь гайд — через интерфейс VS Code (в Cursor всё то же самое, UI идентичен). Терминал не нужен.

---

## Шаг 0. Что нужно установить

### 1. Сам Git

**Mac:**
- Открой Terminal (Cmd+Space → "Terminal").
- Вставь: `git --version` и нажми Enter.
- Если показал версию — git уже стоит, идём дальше.
- Если Mac предложит установить "Command Line Developer Tools" — нажми "Install", подожди 5 минут.

**Windows:**
- Зайди на [git-scm.com/download/win](https://git-scm.com/download/win).
- Скачай установщик, запусти.
- На всех шагах жми "Next" — настройки по умолчанию подходят.
- Перезапусти компьютер.

### 2. VS Code (или Cursor)

Если ещё не стоит:
- VS Code: [code.visualstudio.com](https://code.visualstudio.com)
- Cursor: [cursor.com](https://cursor.com)

### 3. GitHub-аккаунт

- Зарегистрируйся на [github.com](https://github.com) (бесплатно).
- Запомни логин и email — пригодятся на шаге 5.

## Шаг 1. Превращаем папку проекта в гит-репозиторий

1. Открой папку проекта в VS Code: **File → Open Folder**.
2. Слева на панели найди иконку с тремя кружочками на ветке (Source Control). Горячая клавиша: `Ctrl+Shift+G`.
3. Нажми кнопку **Initialize Repository**.

Всё. Папка теперь репозиторий. VS Code начал следить за каждым изменением файла.

**Как понять, что сработало:** рядом с именами файлов появились буквы. `U` — новый файл (untracked), `M` — изменённый (modified), `D` — удалённый (deleted).

---

## Шаг 2. Представься гиту (делается один раз)

Гит должен знать, кто делает изменения. Это не аккаунт, а просто подпись в истории.

В VS Code:

1. Нажми `Cmd+Shift+P` (Mac) / `Ctrl+Shift+P` (Win) — откроется командная палитра.
2. Введи: **Git: Set Username** → Enter → введи имя → Enter.
3. Снова `Cmd+Shift+P` → **Git: Set Email** → введи email, которым регистрировался на GitHub.

Готово, больше к этому возвращаться не надо.

---

## Шаг 3. Первый коммит — сохраняем точку возврата

Коммит = именованная точка в истории проекта. К ней можно вернуться в любой момент.

1. Открой панель Source Control (`Ctrl+Shift+G`).
2. Вверху увидишь список файлов под заголовком **Changes**. Это всё, что изменилось с прошлого коммита.
3. Наведи на заголовок **Changes** — справа появится иконка **+** (Stage All Changes). Нажми её. Файлы переедут в **Staged Changes** — они готовы к сохранению.
4. В поле сверху (там написано "Message") напиши короткое описание: *первая версия проекта* или *добавил форму логина*.
5. Нажми кнопку **Commit** (галочка или синяя кнопка под полем сообщения).

Первая точка сохранения готова. Повторяй это после каждой законченной мысли.

**Правило хорошего коммита:** одна логическая задача = один коммит. Агент сделал фичу — коммит. Починил баг — коммит. Написал тесты — коммит.

---

## Шаг 4. Ветки — песочница для экспериментов

Ветка = параллельная вселенная проекта. В ней можно творить что угодно, основной код не пострадает.

### Как создать ветку

1. Внизу VS Code слева увидишь название текущей ветки (обычно **main**).
2. Нажми на него.
3. Сверху откроется меню — выбери **Create new branch**.
4. Введи имя: `design-experiment` или `try-new-lib`. Без пробелов, через дефис.
5. Enter. Ты в новой ветке.

Всё, что ты теперь коммитишь — идёт только в эту ветку. Main не тронут.

### Как переключаться между ветками

Снова жмёшь на имя ветки внизу слева → выбираешь нужную. VS Code переключит файлы в тот вид, в котором они были в той ветке.

### Как работать параллельно с агентом

Схема для тестирования гипотез:

1. Создал ветку *подход-А*, попросил агента сделать по первому подходу.
2. Переключился на main, создал ветку *подход-Б*, попросил агента сделать по второму.
3. Сравнил результаты, оставил лучший.

### Как удалить ненужную ветку

1. `Cmd+Shift+P` → **Git: Delete Branch**.
2. Выбери ветку из списка.

Экспериментировать можно без страха — оригинал в main всегда на месте.

## Шаг 5. Откат ошибок агента

Вот ради этого всё и затевалось. Разберём три ситуации.

### Ситуация А. Агент напортачил в файле, но ты ещё НЕ сделал коммит

Ты видишь, что файл стал хуже. Откатить его к последней сохранённой версии:

1. В панели Source Control найди файл в списке **Changes**.
2. Наведи на него — справа появится иконка **↶** (Discard Changes).
3. Нажми, подтверди. Файл вернётся к состоянию последнего коммита.

**Осторожно:** это безвозвратно удалит все изменения в файле с момента последнего коммита. Поэтому коммитить надо часто.

### Ситуация Б. Агент сломал всё, и ты НЕ коммитил уже час

Открой Source Control → наведи на **Changes** → иконка **↶** (Discard All Changes). Весь проект вернётся к последнему коммиту. Как будто агента никогда тут не было.

### Ситуация В. Ты уже сделал плохой коммит и хочешь откатить проект на несколько коммитов назад

1. Открой Source Control → внизу открой вкладку **GRAPH**.
2. Увидишь список коммитов. Найди тот, к которому хочешь вернуться (до того, как агент начал ломать).
3. Правой кнопкой → **Checkout (Detached)** чтобы посмотреть, правильный ли это коммит.
4. Если да — создавай новую ветку с места предыдущего "чекпоинта" и работай в ней.

Вместо создания новой ветки можешь использовать `git reset`, но это за пределами данного гайда.

## Шаг 6. Push на GitHub — бэкап и облако

Пока всё, что ты делал, лежит только на твоём ноутбуке. Сольётся диск — потеряешь историю. GitHub решает это.

### Первый раз: связываем проект с GitHub

1. Зайди на [github.com](https://github.com) → кнопка **New** (зелёная, сверху).
2. **Repository name:** название проекта. **Private** — чтобы никто не видел. Больше ничего не трогай. **Create repository**.
3. Вернись в VS Code.
4. Открой Source Control (`Ctrl+Shift+G` / `Cmd+Ctrl+G`).
5. Нажми большую синюю кнопку **Publish Branch**.
6. VS Code попросит залогиниться в GitHub — всплывёт окно браузера, подтверди.
7. Выбери репозиторий, который только что создал. Готово.

Теперь проект в облаке. Открой его на GitHub — увидишь все файлы и историю коммитов.

### Дальше: синхронизация после каждого коммита

После очередного коммита внизу слева появляется иконка с цифрой и стрелками (↑↓). Это значит: у тебя есть коммиты, которых нет на GitHub.

Нажми на неё → **Sync Changes**. Коммиты улетят на GitHub.

Делай это в конце каждой сессии — так у тебя всегда свежий бэкап.

### Как восстановить проект на другом компьютере

1. На новом компьютере: установи VS Code и Git (шаг 0).
2. В VS Code: `Cmd+Shift+P` → **Git: Clone**.
3. Вставь ссылку с GitHub (на странице репозитория кнопка **Code** → скопируй URL).
4. Выбери папку, куда скачать. Готово — весь проект с историей у тебя.

## Чеклист вайбкодера

Делай это каждый день:

- [ ] Начал сессию — убедись, что ты в нужной ветке (видно внизу слева)
- [ ] Хочешь попробовать что-то рискованное — создай ветку
- [ ] Законченная мысль — коммит с описательным именем
- [ ] Агент напортачил — Discard Changes или Reset на хороший коммит
- [ ] Конец сессии — Sync Changes (push на GitHub)

---

## Пять команд для тех, кто всё-таки полез в терминал

Иногда быстрее нажать клавиши, чем кликать. Минимум такой. Сделать папку репозиторием:

```bash
git init
```

Отметить изменения:

```bash
git add .
```

Сохранить коммит (точку сохранения):

```bash
git commit -m "что сделал"
```

Создать и перейти в ветку:

```bash
git checkout -b новая-ветка
```

Вернуться на основную ветку:

```bash
git checkout main
```
