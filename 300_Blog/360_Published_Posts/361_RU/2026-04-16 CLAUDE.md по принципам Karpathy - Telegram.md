# CLAUDE.md по принципам Karpathy

_16 апреля 2026_

---

## Версия для Telegram (RU)

Андрей Карпатый собрал типичные ошибки ИИ при написании кода. Кто-то оформил это в файл CLAUDE.md. За неделю — 15 000 звёзд на GitHub.

Что в нём такого?

ИИ при кодинге делает одни и те же косяки: усложняет архитектуру, игнорирует существующий код, тащит лишние зависимости, «улучшает» то, что не просили. Ошибки предсказуемые — а значит, их можно предотвратить одним файлом.

Идея простая. Кладёшь в корень проекта файл CLAUDE.md с правилами. ИИ читает его при каждом запуске и следует им автоматически. Не нужен фреймворк, не нужен плагин. Один текстовый файл меняет поведение на уровне всего проекта.

Четыре принципа из файла:

1. Сначала думай, потом пиши.
2. Выбирай простое решение.
3. Меняй только то, что просили менять.
4. Проверяй результат, а не процесс.

По сути — здравый смысл. Но ИИ без явных инструкций этот здравый смысл игнорирует.

Цифры: без файла с правилами ИИ нарушал стандарты проекта в 40% случаев. С файлом — в 3%. Настройка заняла 5 минут. Тринадцатикратное снижение ошибок.

И самое приятное — файл не обязательно писать руками. Можно попросить сам ИИ прочитать проект и сгенерировать правила под него. Он видит архитектуру, паттерны, стек и создаёт файл за минуту:

```bash
claude -p "Read the entire project and create a CLAUDE.md based on: Think Before Coding, Simplicity First, Surgical Changes, Goal-Driven Execution. Adapt to the real architecture you see." --allowedTools Bash,Write,Read
```

У кого ИИ пишет код — попробуйте, потом
