# Claude Code папка .claude — тред для Threads

Адаптация статьи [[claude-code-folder-guide]] в формат треда для Threads.

---

## Пост 1 (хук)

Большинство пользователей Claude Code никогда не открывали папку .claude.

А там лежит центр управления. Инструкции, команды, разрешения и даже память между сессиями.

Разложу по частям. Начнём с того, что папка не одна.

---

## Пост 2

Папок .claude на самом деле две.

Одна в проекте — командная конфигурация, коммитится в git. Вся команда работает по одним правилам.

Вторая в домашней директории — ваши личные настройки и история сессий.

Но самое главное лежит внутри проектной.

---

## Пост 3

CLAUDE.md — самый важный файл во всей системе.

Claude читает его первым при запуске сессии. Что напишете — то и будет делать. Всегда.

Команды сборки, архитектурные решения, конвенции именования. Держите до 200 строк, иначе качество падает.

А когда 200 строк станет мало — есть способ масштабировать.

---

## Пост 4

Папка rules/ решает проблему разросшегося CLAUDE.md.

Каждый markdown-файл внутри загружается автоматически. Один файл на тему: api-conventions.md, testing.md, security.md.

Можно привязать правило к конкретным папкам проекта — и оно активируется только при работе с ними.

Но правила это пассивная штука. А вот команды — уже интереснее.

---

## Пост 5

Папка commands/ превращает файлы в слеш-команды.

Положили review.md — получили /project:review. Внутри можно вставлять вывод shell-команд. Например, автоматически подставить git diff в промпт для ревью.

Команды запускаете вы. А есть штука, которую Claude запускает сам.

---

## Пост 6

Skills — навыки, которые Claude вызывает без вашей команды.

Описываете в SKILL.md когда применять. Говорите "проверь на уязвимости" — Claude сам находит нужный навык и запускает.

Навык это пакет файлов. Команда — один файл. Навык — целая папка с документацией.

Но есть ещё более мощная штука — агенты.

---

## Пост 7

Agents — выделенные субагенты со своим контекстом.

Каждый со своей моделью, набором инструментов и системным промптом. Работают изолированно — не засоряют основную сессию.

Security-аудитор с доступом только на чтение. Code-reviewer на дешёвой модели Haiku.

Осталось разобраться, как ограничить то, что Claude может делать.

---

## Пост 8

settings.json контролирует разрешения.

allow — выполняется без вопросов.
deny — заблокировано полностью.
Всё остальное — Claude спросит перед выполнением.

Минимум: разрешите ваши run-команды, запретите чтение .env.

Теперь собираем всё вместе.

---

## Пост 9 (закрытие)

План настройки за 5 шагов:

Запустите /init. Настройте settings.json. Создайте пару команд. Разделите CLAUDE.md на rules/. Добавьте глобальный ~/.claude/CLAUDE.md.

CLAUDE.md — файл с максимальным рычагом. Начните с него. Всё остальное оптимизация.

А вы уже копались в своей папке .claude?
