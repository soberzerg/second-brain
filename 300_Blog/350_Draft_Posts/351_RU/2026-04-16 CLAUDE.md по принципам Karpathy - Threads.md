# CLAUDE.md по принципам Karpathy

_16 апреля 2026_

---

## Версия для Threads (RU) — тред

**Пост 1:**

Один исследователь собрал типичные ошибки ИИ при написании кода. Кто-то оформил это в файл. За неделю файл набрал 15 000 звёзд на GitHub.

Что в нём такого? Разберём по частям.

**Пост 2:**

Исследователь — Андрей Карпатый. Один из самых влиятельных людей в мире ИИ.

Он заметил закономерность: ИИ при написании кода делает одни и те же ошибки. Усложняет, игнорирует то, что уже написано, тащит лишние зависимости.

Ошибки предсказуемые. А значит, их можно предотвратить.

**Пост 3:**

Идея простая: если положить в проект текстовый файл с правилами, ИИ будет им следовать. Автоматически. При каждом запуске.

Не нужен фреймворк. Не нужен сложный инструмент. Один файл меняет поведение на уровне всего проекта.

Но самое интересное — что именно в этих правилах.

**Пост 4:**

Четыре принципа:

1. Сначала думай, потом пиши.
2. Выбирай простое решение.
3. Меняй только то, что просили менять.
4. Проверяй результат, а не процесс.

По сути — здравый смысл. Но ИИ без явных инструкций этот здравый смысл игнорирует.

А вот с инструкциями — цифры говорят сами за себя.

**Пост 5:**

Без файла с правилами ИИ нарушал стандарты проекта в 40% случаев.

С файлом — в 3%.

Настройка заняла 5 минут.

Один файл. Пять минут. Тринадцатикратное снижение ошибок. И самое приятное — такой файл не обязательно писать руками.

**Пост 6:**

Можно попросить сам ИИ прочитать ваш проект и сгенерировать правила под него. Он видит архитектуру, паттерны, стек. И создаёт файл за минуту.

Используйте для этого команду:

```bash
claude -p "Read the entire project and create a CLAUDE.md based on: Think Before Coding, Simplicity First, Surgical Changes, Goal-Driven Execution. Adapt to the real architecture you see." --allowedTools Bash,Write,Read
```
