Готово. Вот результат от Клода — уже пригодный к публикации.

Основной тред

1/
Anthropic тихо выпустил фичу, после которой у “AI агентов” заканчиваются отговорки.

Routines в Claude Code.

Это больше не AI-помощник, которого ты зовёшь. Это агент, который живёт и работает сам — пока ты спишь.

2/
Что это такое, если коротко.

Настраиваешь один раз: промпт, репозиторий, коннекторы.

Дальше оно запускается по расписанию, по API-вызову или по событию. Без тебя в кадре.

3/
Ключевой момент, который меняет всё.

Всё крутится на облачной инфре Anthropic.

Не надо держать ноут открытым. Не надо следить за сессией, токенами, процессом. Задал расписание — агент работает.

4/
До этого Claude Code был интерактивным инструментом. Открыл терминал, поработал, закрыл.

Теперь это фоновая автоматизация.

AI перестаёт быть “режимом работы” и становится чем-то вроде инфраструктуры. Сервисом, а не приложением.

5/
Сценарий 1: ночной уборщик в репозитории.

В 3 ночи Routine проходится по багам, чинит тривиальные вещи, открывает PR.

Утром ты приходишь ревьюить, а не писать.

6/
Сценарий 2: дежурный по инбоксу задач.

Каждый час агент разбирает issues, вешает лейблы, бьёт задачи на подзадачи, синхронизирует с трекером.

Ты не видишь шум. Ты видишь готовую картину.

7/
Сценарий 3: реакция на событие.

Приходит webhook — агент просыпается, собирает контекст, делает изменение, пушит.

Это уже не ассистент. Это worker.

8/
Сравни со старым режимом.

Раньше: “запусти Claude Code у себя, не закрывай терминал, следи”.
Теперь: “положи задачу в расписание и забудь”.

Разница такая же, как между локальным скриптом и cron job на сервере.

9/
Что это значит для рынка AI агентов 👇

Два года все говорили про агентов. Но 90% фреймворков жили внутри ноутбука разработчика и ломались, как только он закрывал крышку.

10/
Routines — первый нормальный шаг к тому, чтобы агенты работали как сервисы.

Без DIY-оркестраторов. Без “поставь mac mini под стол”. Без костылей.

Это не фича. Это сдвиг модели.
