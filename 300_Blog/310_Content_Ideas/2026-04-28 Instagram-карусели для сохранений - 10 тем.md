# 10 тем для Instagram-каруселей на сохранения

> Финальный отбор из пула кандидатов. Критерии: высокая ожидаемая сохраняемость + практическая ценность для аудитории «соло-фаундер с ИИ-рычагом / вайбкодер». Покрыты все 4 формата: чек-лист, пошаговая инструкция, подборка, сравнение.

## Принципы отбора

Темы для **сохранений** — это reference-контент, к которому подписчик хочет вернуться. Сильнее всего работают:

1. **Конкретные числа в заголовке** («7 skills», «10 промптов», «12 пунктов») — сразу обещают законченную единицу полезного.
2. **Решающие фреймворки** («когда использовать что») — снимают когнитивную нагрузку выбора.
3. **Названные инструменты** — без воды, по делу.
4. **Применимость в течение часа** после прочтения — иначе сохранят и забудут, охват упадёт.

Каждая тема ниже опирается на собственный опыт и материалы, уже накопленные в этом vault — это даёт «несгораемое» наполнение и снижает риск AI-блёклости.

## Финальные 10 тем

### 1. Чек-лист: 12 пунктов перед запуском MVP в одиночку

- **Формат:** чек-лист
- **Слайды:** обложка + 12 пунктов (домен, лендинг, аналитика, оплаты, стартовая аудитория, первое CustDev-интервью и т. д.) + слайд «сохрани, пройдись по списку перед запуском».
- **Почему вошла:** универсальная боль соло-фаундера, прямой trigger на сохранение («пригодится через неделю»). Опирается на материалы 100_Tasks и опыт AGIents/AGIency.

### 2. Чек-лист: что положить в CLAUDE.md (10 обязательных секций)

- **Формат:** чек-лист
- **Слайды:** обложка + 10 секций (структура проекта, команды, тестирование, code style, security, hooks, agents, performance и т. д.) + «забрать готовый шаблон».
- **Почему вошла:** ниша вайбкодинга у автора уже прогрета (посты «CLAUDE.md по принципам Karpathy»), у CLAUDE.md высокая частота поиска и нулевой канон — карусель становится «справочником».

### 3. Как написать свой первый Claude skill за 30 минут

- **Формат:** пошаговая инструкция
- **Слайды:** обложка → проблема (skill ≠ агент) → 6 шагов: определить триггер, написать SKILL.md, добавить ресурсы, проверить через `/list`, протестировать, опубликовать → итог + CTA.
- **Почему вошла:** прямое продолжение «Лайфхак №6 Skills», который уже подтвердил спрос. Пошаговый формат с воспроизводимым результатом → высокая сохраняемость.

### 4. Как запустить Ralph Loop на ночь: 6 шагов до автономного кодинга

- **Формат:** пошаговая инструкция
- **Слайды:** обложка → что такое Ralph Loop (Geoffrey Huntley, 2025) → 6 шагов: задача в bug.md, harness, allowlist, лимит итераций, мониторинг через лог, утренний review → результат «проснулся — код готов».
- **Почему вошла:** хайповая тема (анонс AgentOS уже в опубликованных постах), редкий формат «пошагово как настроить» — такие сохраняют, чтобы повторить дома.

### 5. Как построить систему памяти агента: 4 уровня хранения

- **Формат:** пошаговая инструкция
- **Слайды:** обложка → провокация «проблема не в памяти, а в системе управления» → 4 уровня: working memory (контекст сессии) → episodic (логи) → semantic (knowledge base / wiki) → procedural (skills/агенты) → как реализовать у себя.
- **Почему вошла:** опирается на свежий тред «Память для агентов и Ralph Loop» (494 знака, уже отработано). Системная схема, к которой возвращаются — топ-формат для сохранений.

### 6. 7 Claude Code skills, которые я использую каждый день

- **Формат:** подборка
- **Слайды:** обложка → 7 skills с одним предложением о пользе каждого (например: ingest, social-content, brainstorming, daily-review, prd, web-research, social-research) → итог + ссылка на репо/гайд.
- **Почему вошла:** «список инструментов от практика» — классический save-magnet, опирается на реальный набор из `~/.claude/skills/`. Прямой kvoting cross-promotion с лид-магнитом.

### 7. 10 промптов соло-фаундера: от валидации идеи до первой продажи

- **Формат:** подборка
- **Слайды:** обложка → 10 промптов с краткой задачей под каждым (валидация боли, ICP-портрет, 47 холодных сообщений, скрипт discovery-звонка, voicemail, follow-up, оффер, кейс, отзыв, эпистолярный pricing) → «сохрани, чтобы не писать с нуля».
- **Почему вошла:** «10 промптов» — один из самых сохраняемых форматов в нише. Соответствует фокусу блога (Founder Bootstrapping) и закрывает реальные шаги воронки.

### 8. 5 ИИ-агентов, которые экономят соло-фаундеру 10 часов в неделю

- **Формат:** подборка
- **Слайды:** обложка → 5 агентов с конкретной ролью (research, inbox-processor, daily-review, planner, code-reviewer) → метрика времени на каждом → итог «10+ часов в неделю освобождают фокус».
- **Почему вошла:** конкретные часы + конкретные агенты = «сохрани и попробуй у себя». Отвечает на популярный поисковый запрос «как ИИ заменяет команду».

### 9. Claude Opus vs Sonnet vs Haiku: когда какую модель использовать

- **Формат:** сравнение
- **Слайды:** обложка → таблица 3×N (стоимость / скорость / задача / контекст / типичный кейс) → правило выбора (Haiku — массовые рутины, Sonnet — основная разработка, Opus — архитектура и ресёрч) → шпаргалка.
- **Почему вошла:** decision-matrix для дорогого решения = один из самых сохраняемых форматов в Instagram tech-нише. Опирается на материал из rules/common/performance.md.

### 10. Cursor vs Claude Code vs Aider: какой инструмент выбрать вайбкодеру

- **Формат:** сравнение
- **Слайды:** обложка → 4 критерия (UX/IDE, агентность, цена, кривая обучения) → таблица → вердикт по сценариям («только пишу скрипты», «веду продакшн-проект», «кодю с фронта на iPad»).
- **Почему вошла:** прямое сравнение трёх горячих инструментов = высокая сохраняемость («посмотрю, когда буду выбирать»). Бьёт в позиционирование «вайбкодинг-эксперт».
