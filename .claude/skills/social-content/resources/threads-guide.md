# Гайд по Threads для AISobolev

## Обзор платформы

- **Лимит:** 500 символов на пост
- **Язык:** Русский
- **Формат:** Plain text (без markdown — Threads не поддерживает форматирование)
- **Хештеги:** НЕ использовать — засоряют текст и снижают восприятие
- **Сниппеты кода:** НЕ использовать — аудитория Threads не техническая
- **Роль в стратегии:** Ежедневные короткие мысли, engagement, привлечение новой аудитории

---

## Правила форматирования

- **Plain text только** — без **bold**, без *italic*, без `code`, без H2/H3
- **Одна мысль = один пост** — не пытайся впихнуть три идеи в 500 символов
- **Эмодзи как пунктуация** — допустимы, но 1-2 максимум (не декоративно)
- **Короткие предложения** — 5-10 слов на предложение
- **Без хештегов** — не добавлять хештеги ни в текст, ни в конец
- **Без кода** — никаких сниппетов, команд терминала, конфигов

## Типы постов

### 1. Микро-инсайт

Одно наблюдение из жизни фаундера. Короткое, конкретное, неожиданное.

**Пример:**
```
Отправил 47 холодных сообщений.
Ответили 2 человека.
Один стал клиентом на $500/мес.

Маркетинг стоимостью 0 рублей.
```

### 2. Behind-the-scenes

Что только что произошло — в реальном времени.

**Пример:**
```
Только что закрыл месяц.
Выручка: $2100.
Расходы: $340.
Чистыми: $1760.

Год назад в найме получал меньше.
```

### 3. Вопрос / обсуждение

Провокационный вопрос, который запускает дискуссию в комментах.

**Пример:**
```
Вопрос к фаундерам:
Что вы сделали ПЕРВЫМ — продукт или продажу?

Я начал с продажи. И это было лучшее решение.

А вы?
```

### 4. Hot take

Короткое провокационное утверждение.

**Пример:**
```
Бизнес-план — это документ, который никто не читает после написания.

Включая тебя.
```

### 5. Тред (серия постов)

Серия из 5-10 постов, связанных одной темой. **Главная метрика — глубина прочтения.** Каждый пост должен затягивать в следующий.

Подробные правила — в секции "Правила тредов" ниже.

---

## Правила тредов (КРИТИЧНО)

### Принцип Slippery Slide

Каждый пост в треде заканчивается "крючком" — тизером следующего. Читатель не может остановиться, потому что последняя строка обещает развитие.

**Техника:** последнее предложение каждого поста (кроме финального) должно создавать незакрытую петлю — обещание чего-то более интересного, важного или неожиданного в следующем посте.

### Типы крючков

- **Обещание эскалации:** "Но это ещё не самое интересное."
- **Тизер темы:** "А вот дальше — то, ради чего всё затевалось."
- **Контраст:** "Команды решают одну проблему. Но есть штука мощнее."
- **Переход от пассивного к активному:** "Правила — пассивная штука. А вот команды — уже интереснее."
- **Вопрос-мост:** "Осталось разобраться, как это ограничить."
- **Сборка:** "Теперь собираем всё вместе."

### Нарастание интенсивности

Крючки должны идти по нарастающей — от мягких к сильным. Нельзя начать с "самое мощное" и потом скатиться в "а ещё вот это".

**Пример прогрессии:**
1. "Но самое главное лежит внутри."
2. "А когда этого станет мало — есть способ масштабировать."
3. "Но правила — пассивная штука. А вот команды — уже интереснее."
4. "Команды запускаете вы. А есть штука, которую система запускает сама."
5. "Но есть ещё более мощная штука."
6. "Осталось разобраться, как всё это ограничить."
7. "Теперь собираем всё вместе."

### Структура треда

**Пост 1 (хук):** Зацепить + обозначить тему + первый крючок. НЕ использовать "Тред 👇" — вместо этого крючок к следующему посту.

**Посты 2-N (содержание):** Один пункт на пост. Последняя строка — крючок к следующему.

**Финальный пост (закрытие):** Вывод + вопрос к аудитории.

### Пример треда

**Пост 1:**
```
Большинство пользователей никогда не открывали настройки своего инструмента.

А там лежит центр управления. Инструкции, команды, разрешения.

Разложу по частям. Начнём с того, что настроек не одна, а две.
```

**Пост 2:**
```
Первая — общая для команды. Коммитится в репозиторий.
Вторая — ваши личные предпочтения.

Но самое главное лежит внутри командной.
```

**Пост 3:**
```
Главный файл — инструкция для системы.

Что напишете — то она и будет делать. Всегда.

А когда инструкция разрастётся — есть способ масштабировать.
```

**Пост N (финал):**
```
Весь план настройки за 5 шагов:

Инициализация. Разрешения. Пара команд. Модульные правила. Личные настройки.

Главный файл — с максимальным рычагом. Начните с него.

А вы уже копались в настройках?
```

### Чего НЕ делать в тредах

- ❌ **"Тред 👇"** — заменять крючком к следующему посту
- ❌ **Нумерация постов** ("1/7", "Пост 3") — ломает естественность
- ❌ **Одинаковые крючки** — каждый должен быть уникальным
- ❌ **Крючок без содержания** — если пост пустой, крючок не спасёт
- ❌ **Затухающая интенсивность** — если крючки слабеют к концу, тред не дочитают

```
Клиент написал: "А можно ещё вот это?"
```

```
Самый полезный навык фаундера — не кодинг.
И не маркетинг.
А умение продавать в личных сообщениях.

```

```
ИИ позволяет одному человеку делать работу команды.

Большая разница.
```

```

Результат: больше клиентов чем за год нетворкинга.
```

## Engagement-тактики

- **Ответы в первые 30 минут** — алгоритм ценит раннее вовлечение
- **Отвечайте на каждый комментарий** — это удваивает вовлечённость
- **Задавайте вопросы в конце** — провоцируйте обсуждение
- **Время публикации:** 9-10 утра, 12-13 дня, 19-21 вечера (МСК)
- **Треды** — получают больший охват, чем одиночные посты
- **Регулярность** — ежедневный пост тренирует алгоритм

---

## Чего НЕ делать

- ❌ **Кросс-пост из Telegram** — копировать длинный пост и обрезать до 500 символов
- ❌ **Markdown** — Threads не поддерживает, текст будет выглядеть ломаным
- ❌ **Стены текста** — если текст плотный, читатель скроллит дальше
- ❌ **Хештеги** — засоряют текст, снижают восприятие
- ❌ **Сниппеты кода** — аудитория Threads не техническая
- ❌ **Generic мотивация** — "Верь в себя и всё получится" ≠ founder-контент
- ❌ **Промо без ценности** — "Подписывайтесь на мой канал" без контекста
