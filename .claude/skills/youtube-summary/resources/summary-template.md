# YouTube Summary Template

Используй этот шаблон для создания всех выжимок из YouTube видео.

## Frontmatter

```yaml
---
tags:
  - youtube
  - выжимка
  - [дополнительные теги по теме видео]
source: "[URL видео]"
date: [текущая дата в формате YYYY-MM-DD]
video_title: "[Точное название видео]"
---
```

## Правила форматирования

### Название файла

- **Формат:** `YYYY-MM-DD - [Название видео].md`
- **Очистка:** Убрать символы `/ \ : * ? " < > |`
- **Длина:** Макс 50 символов для названия (после даты), иначе обрезать + `...`

### Язык и термины

- **Основной язык:** Русский
- **Технические термины:** На английском как есть (API, RAG, MCP, CI/CD, etc.)
- **Имена:** Как произносятся (David Ogilvy → Дэвид Огилви, но оставлять оригинал в скобках при первом упоминании)

### Стиль написания

- **Конкретика:** Избегать "автор рассказывает о важности...", писать ЧТО именно говорит
- **Контекст:** Для технических деталей указывать в каком контексте упоминается
- **Примеры:** Включать если автор приводит кейсы или примеры
- **Цитаты:** Можно использовать прямую речь если формулировка важна

### Теги

Автоматически добавлять теги:
- `youtube` - всегда
- `выжимка` - всегда
- По теме видео (например: `ai`, `бизнес`, `разработка`, `маркетинг`, `продакт`)

### Что НЕ включать

- ❌ Пересказ каждой минуты видео
- ❌ Общие фразы без конкретики
- ❌ Информацию из описания видео (только из контента)
- ❌ Рекламные интеграции (если они не важны для темы)
- ❌ Призывы подписаться/лайкнуть

### Что ОБЯЗАТЕЛЬНО включать

- ✅ Точное название видео
- ✅ URL источника
- ✅ Основную тему в 1-2 предложениях
- ✅ Ключевые мысли автора
- ✅ Упомянутые ресурсы (книги, инструменты, люди)

# Выжимка: Building AI Agents with Claude MCP

**Источник:** https://youtube.com/watch?v=example123
**Дата:** 2026-02-10

## Основная тема

Практическое руководство по созданию ИИ-агентов с использованием Claude и Model Context Protocol (MCP). Автор показывает архитектуру multi-agent систем и best practices для production-ready решений.

## Ключевые тезисы

1. **MCP как стандарт расширения агентов** - MCP позволяет агентам подключать внешние инструменты без изменения кода агента. Это решает проблему vendor lock-in.

2. **Архитектура orchestrator + worker agents** - Один главный агент (orchestrator) координирует специализированных агентов (workers). Каждый worker отвечает за свою область: поиск данных, анализ, генерация контента.

3. **Важность prompt engineering** - 80% качества агента зависит от промпта. Автор рекомендует структуру: Role → Context → Task → Constraints → Examples.

[... ещё 5-7 тезисов ...]

## Полезные инсайты

- **MCP servers можно переиспользовать** - Один MCP server (например, для работы с Google Drive) может использоваться разными агентами. Это экономит время на разработку.

- **Stateless агенты проще масштабировать** - Если агент не хранит состояние между запросами, его легче deploy в serverless окружении.

[... ещё 2-3 инсайта ...]

## Практические рекомендации

- [ ] **Начинать с одного агента** - Не усложнять архитектуру раньше времени. Multi-agent системы нужны когда один агент не справляется.

- [ ] **Тестировать на реальных данных** - Synthetic данные не покажут edge cases, которые будут в production.

- [ ] **Логировать все взаимодействия** - Для отладки критически важно видеть полную цепочку вызовов агента.

## Упомянутые ресурсы

- **LangChain** (фреймворк) - Используется для создания цепочек вызовов LLM, но автор не рекомендует для простых кейсов
- **Anthropic Claude API** (API) - Основа для агентов, поддержка function calling
- **Model Context Protocol** (спецификация) - Открытый стандарт от Anthropic для расширения агентов
