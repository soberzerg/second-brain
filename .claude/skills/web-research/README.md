# Web Research Skill

Комплексный навык для веб-исследований: парсинг страниц, поиск информации, анализ конкурентов, сбор кейсов.

## Возможности

- 🔍 **Поиск информации** - найти статьи, документацию, новости по теме
- 📄 **Парсинг страниц** - извлечь контент с любого URL
- 🔬 **Анализ конкурентов** - исследовать конкурирующие продукты
- 📊 **Сбор кейсов** - найти примеры применения технологий
- 📈 **Мониторинг трендов** - отслеживать новости индустрии
- 📸 **Скриншоты страниц** - сохранять визуальный контент

## Быстрый старт

### Поиск информации

```text
Найди информацию про Claude MCP protocol
```

Навык автоматически:

1. Найдёт топ-5 источников через Firecrawl
2. Извлечёт ключевые моменты
3. Сохранит в `500_Research/580_Web_Research/`
4. Предложит углубиться в конкретный источник

### Парсинг страницы

```text
Спарси эту страницу: https://example.com/article
```

Навык автоматически:

1. Извлечёт контент страницы
2. Определит: title, author, date, content
3. Покажет краткое резюме
4. Сохранит с правильным frontmatter

### Анализ конкурентов

```text
Проанализируй конкурентов для AGIents.pro
```

Навык автоматически:

1. Найдёт 5-7 конкурентов
2. Извлечёт: название, URL, фичи, ценообразование
3. Создаст сравнительную таблицу
4. Выделит gaps и opportunities
5. Предложит brainstorming для стратегии

### Сбор кейсов

```text
Собери кейсы использования AI agents в B2B
```

Навык автоматически:

1. Найдёт релевантные кейсы
2. Структурирует: компания, задача, решение, результаты
3. Выделит общие паттерны
4. Предложит создать пост (post-writing)

## Используемые инструменты

### Firecrawl (основной)

- `firecrawl_search` - поиск информации
- `firecrawl_scrape` - парсинг страницы
- `firecrawl_agent` - автономные исследования
- `firecrawl_extract` - структурированное извлечение

### Playwright (для динамических страниц)

- Автоматический fallback если Firecrawl не справился
- Скриншоты страниц
- Взаимодействие с элементами

## Сохранение результатов

### Пути

- **Общие исследования**: `500_Research/580_Web_Research/`
- **Проектные исследования**: `200_Projects/[Проект]/Research/`
- **Конкуренты**: `200_Projects/[Проект]/Research/Competitors/`

### Naming Convention

```text
YYYY-MM-DD - [Тема].md
```

Примеры:

- `2026-02-10 - AI Agents Production Best Practices.md`
- `2026-02-10 - AGIents Competitor Analysis.md`
- `2026-02-10 - MCP Protocol Deep Dive.md`

### Frontmatter

```yaml
---
type: web-research-[article|competitors|case|trends]
date: YYYY-MM-DD
source_url: [URL]
tags: [тег1, тег2]
projects: [[связанный проект]]
---
```

## Tool Selection

Навык автоматически выбирает оптимальный инструмент:

1. **firecrawl_search** - если нет точного URL
2. **firecrawl_scrape** - если есть конкретный URL
3. **firecrawl_agent** - если сложная задача
4. **firecrawl_extract** - если нужна структура
5. **Playwright** - если динамический контент

См. детали в [SKILL.md](SKILL.md)

## Параметры оптимизации

### Для скорости

- `maxAge: 172800000` - кеш на 48 часов
- `limit: 5-10` - оптимальное количество для search

### Для качества

- `onlyMainContent: true` - убрать навигацию
- `formats: ["markdown"]` - удобный формат

## Edge Cases

### Страница недоступна

```markdown
❌ Не удалось получить доступ

Возможные проблемы:
• Требуется авторизация
• Блокировка скрейпинга
• 404 - страница не найдена

Что делать:

1. Проверь URL в браузере
2. Попробуй другой источник
3. Используй firecrawl_agent
```

### Динамический контент

```markdown
⚠️ Firecrawl не смог полностью загрузить страницу

Переключаюсь на Playwright для полного рендеринга...
```

### Нет результатов

```markdown
🤷 Не найдено результатов

Попробуй:

1. Изменить формулировку
2. Использовать общие термины
3. Поиск на английском
```

## FAQ

### Как навык выбирает инструмент?

1. Если есть URL - `firecrawl_scrape`
2. Если нет URL - `firecrawl_search`
3. Если сложная задача - `firecrawl_agent`
4. Если не сработало - Playwright

### Куда сохраняются результаты?

- По умолчанию: `500_Research/580_Web_Research/`
- Если связано с проектом: `200_Projects/[Проект]/Research/`
- Автоматически определяется из контекста

### Можно ли использовать без команды?

Да! Навык активируется автоматически при ключевых словах:

- "Найди информацию про ..."
- "Спарси ..."
- "Проанализируй конкурентов ..."

### Как работает интеграция с другими навыками?

После завершения research навык проактивно предлагает:

- **post-writing** - создать пост на основе данных
- **brainstorming** - обсудить стратегию
- Не активирует автоматически, только предлагает

### Что если Firecrawl не справляется?

Автоматический fallback на Playwright для:

- Динамических SPA (React, Vue, Angular)
- JavaScript-heavy сайтов
- Когда нужны скриншоты
