# Web Research Skill

## When to use this skill

**Автоматическая активация при:**

- Запросы на парсинг: "спарси", "scrape", "extract from URL"
- Исследования конкурентов: "анализ конкурентов", "что делают [компания]"
- Поиск информации: "найди информацию про", "research", "исследуй"
- Сбор данных: "собери примеры", "найди кейсы"

**Паттерны активации:**

- "Спарси эту страницу: [URL]"
- "Найди информацию про [тема]"
- "Проанализируй конкурентов [компания]"
- "Собери кейсы по [тема]"
- "Что говорят про [тема] в интернете"
- "Исследуй [URL] и извлеки [данные]"

## Workflow

### 1. **Определение типа запроса**

   - Single page scraping - парсинг одной страницы
   - Multi-page research - исследование нескольких источников
   - Competitor analysis - анализ конкурентов
   - Trend monitoring - мониторинг трендов
   - Data aggregation - агрегация данных

### 2. **Выбор инструмента**

   - Firecrawl search - если нужен поиск
   - Firecrawl scrape - если есть конкретный URL
   - Firecrawl agent - если задача сложная и нет точных URL
   - Firecrawl extract - если нужны структурированные данные
   - Playwright - если динамический контент или нужны скриншоты

### 3. **Выполнение исследования**

   - Запустить соответствующий инструмент
   - Обработать результаты
   - Извлечь ключевую информацию

### 4. **Структурирование результата**

   - Создать markdown заметку с результатами
   - Добавить метаданные и источники
   - Связать с проектами если релевантно

### 5. **Сохранение в vault**

   - По умолчанию: `500_Research/580_Web_Research/`
   - Или в проект: `200_Projects/[Проект]/Research/`

## Instructions

### Core Principles

- **Точность источников**: Всегда указывать URL и дату извлечения
- **Структурированность**: Организовывать данные в читаемом формате
- **Контекст**: Связывать с проектами из CLAUDE.md
- **Эффективность**: Выбирать оптимальный инструмент для задачи
- **Проверяемость**: Сохранять ссылки на все источники

### Tool Selection Matrix

#### Когда использовать firecrawl_search:

✅ Нужно найти информацию по теме
✅ Не знаешь точные URL
✅ Нужно несколько источников
✅ Поиск новостей, статей, кейсов

Пример:

```typescript
mcp__firecrawl__firecrawl_search({
  query: "AI agents production best practices 2026",
  limit: 5,
  sources: [{type: "web"}],
  scrapeOptions: {
    formats: ["markdown"],
    onlyMainContent: true
  }
})
```

#### Когда использовать firecrawl_scrape:

✅ Есть конкретный URL
✅ Нужен контент одной страницы
✅ Парсинг статьи, блога, документации
✅ Быстрое извлечение контента

Пример:

```typescript
mcp__firecrawl__firecrawl_scrape({
  url: "https://example.com/article",
  formats: ["markdown"],
  onlyMainContent: true,
  maxAge: 172800000  // кеш на 48 часов
})
```

#### Когда использовать firecrawl_agent:

✅ Сложная задача исследования
✅ Не знаешь точно где искать
✅ Нужна агрегация из нескольких источников
✅ Автономное исследование темы

Пример:

```typescript
mcp__firecrawl__firecrawl_agent({
  prompt: "Find top 5 AI startups founded in 2024 with their funding amounts",
  schema: {
    type: "object",
    properties: {
      startups: {
        type: "array",
        items: {
          type: "object",
          properties: {
            name: {type: "string"},
            funding: {type: "string"},
            founded: {type: "string"}
          }
        }
      }
    }
  }
})
```

#### Когда использовать firecrawl_extract:

✅ Нужны структурированные данные
✅ Есть schema для извлечения
✅ Парсинг конкретных полей со страниц
✅ Множественные URL с одинаковой структурой

Пример:

```typescript
mcp__firecrawl__firecrawl_extract({
  urls: ["https://example.com/product1", "https://example.com/product2"],
  prompt: "Extract product name, price, and description",
  schema: {
    type: "object",
    properties: {
      name: {type: "string"},
      price: {type: "number"},
      description: {type: "string"}
    }
  }
})
```

#### Когда использовать Playwright:

✅ Динамический контент (React, Vue, Angular SPA)
✅ Нужны скриншоты страниц
✅ Требуется взаимодействие (клики, формы)
✅ JavaScript-heavy сайты, которые Firecrawl не может спарсить

**Fallback стратегия:**

1. Первая попытка - всегда Firecrawl (быстро, надёжно)
2. Если не сработало - Playwright navigate + evaluate
3. Для визуального контента - screenshot

### Step-by-Step Execution

#### Step 1: Распознать намерение

Определить тип запроса:

- **Search & Research**: "найди информацию", "исследуй тему"
- **Single Page**: конкретный URL с просьбой извлечь контент
- **Competitor Analysis**: "анализ конкурентов", "что делает [компания]"
- **Data Collection**: "собери примеры", "найди кейсы"
- **Trend Monitoring**: "что говорят про", "мониторинг упоминаний"

#### Step 2: Выбрать инструмент

Использовать Tool Selection Matrix выше.

**Fallback стратегия:**

1. Начать с firecrawl_search если нет точного URL
2. Если search не дал результата - использовать firecrawl_agent
3. Если есть URL - всегда использовать firecrawl_scrape
4. Для структурированных данных - firecrawl_extract
5. Если Firecrawl не справился - Playwright

#### Step 3: Выполнить запрос

Параметры для оптимизации:

- `maxAge: 172800000` (48 часов) - для ускорения повторных запросов
- `onlyMainContent: true` - убрать навигацию и footer
- `formats: ["markdown"]` - для большинства случаев
- `limit: 5-10` - для search, больше если нужен глубокий анализ

#### Step 4: Обработать результаты

Извлечь из результата:

- Основной контент
- Метаданные (title, date, author)
- Ссылки на источники
- Ключевые инсайты

#### Step 5: Структурировать и сохранить

Создать markdown файл по шаблону из `resources/extraction-templates.md`

**Формат имени файла:**

- Формат: `YYYY-MM-DD - [Тема исследования].md`
- Пример: `2026-02-10 - AI Agents Production Best Practices.md`

**Путь сохранения:**

- По умолчанию: `500_Research/580_Web_Research/`
- Если связано с проектом: `200_Projects/[Проект]/Research/`

**Определение проекта:**

- AGIents.pro - если про AI agents, no-code platforms, агенты
- Clario - если про ИИ-аналитику продаж, B2B analytics
- AGIency - если про кейсы автоматизации, клиентские проекты
- Express Security Audit - если про код-ревью, безопасность, аудит
- AISobolev блог - если материал для поста

#### Step 6: Подтверждение пользователю

```markdown

**"[Тема исследования]"**

Хочешь углубиться в какой-то из источников?
```

### Edge Cases

#### Страница недоступна

```markdown
❌ Не удалось получить доступ к странице

**URL:** [URL]
**Причина:** [описание ошибки]

Возможные проблемы:
• Страница требует авторизацию
• Сайт блокирует скрейпинг
• 404 - страница не найдена
• Timeout - сайт слишком медленный

Что делать:

1. Проверь URL в браузере
2. Попробуй другой источник
3. Используй firecrawl_agent для автоматического поиска альтернатив
```

#### Слишком много результатов

```markdown
⚠️ Найдено очень много результатов ([X])

Это может занять много времени и места.

Рекомендую:

1. Уточнить запрос (добавить фильтры)
2. Ограничить количество источников
3. Использовать firecrawl_agent для автоматической фильтрации

Продолжить с текущими параметрами или уточнить?
```

#### Нет результатов

```markdown
🤷 Не найдено релевантных результатов

**Запрос:** [запрос]

Попробуй:

1. Изменить формулировку запроса
2. Использовать более общие термины
3. Проверить правописание
4. Попробовать поиск на английском

Хочешь попробовать другой запрос?
```

#### Firecrawl не справился - используем Playwright

```markdown
⚠️ Firecrawl не смог полностью загрузить страницу (динамический контент)

Переключаюсь на Playwright для полного рендеринга...
⏳ Это может занять дольше времени

[... используем Playwright ...]
```

### Tone & Style

- **Автоматизм**: Минимум вопросов, максимум действий
- **Прозрачность**: Показывать что делаешь в процессе
- **Проактивность**: Предлагать следующие шаги
- **Эффективность**: Выбирать оптимальный инструмент

**Избегать:**

- ❌ "Какой инструмент использовать?" (выбери сам)
- ❌ Показывать технические детали API
- ❌ Спрашивать про формат сохранения если он очевиден

**Предпочитать:**

- ✅ "🔍 Исследую..." (сразу к делу)
- ✅ "✅ Найдено 5 источников" (конкретно)
- ✅ "Углубиться в первый источник?" (проактивно)
