# Brainstorming Skill

Навык для стратегического мышления и обсуждения идей.

## Назначение

Автоматически активируется, когда вы:
- Спрашиваете мнение ("как думаешь...")
- Просите поразмышлять ("давай обсудим...")
- Хотите обсудить стратегию или подход
- Нужен совет по решению

## Что умеет

### 1. Анализ бизнес-идей
- Валидация через proven frameworks (JTBD, Trend+Need+Tech, 10x vs 10%)
- Оценка timing и market fit
- Выявление рисков и возможностей

### 2. Технические решения
- Trade-offs анализ
- Build vs Buy vs Partner
- Оценка второго порядка эффектов (second-order effects)

### 3. Решение проблем
- 5 Whys для поиска root cause
- Inversion thinking
- Constraint removal

### 4. Стратегические выборы
- Анализ приоритетов
- Баланс между competing goals
- Долгосрочные vs краткосрочные решения

## Стиль работы

### Что получите
- ✅ Прямые, честные ответы
- ✅ Конкретные рекомендации
- ✅ Несколько вариантов с плюсами/минусами
- ✅ Контекст ваших проектов (из CLAUDE.md)
- ✅ Вопросы для углубления

### Чего НЕ будет
- ❌ "Зависит от ситуации" без конкретики
- ❌ Списков из 10+ пунктов
- ❌ Абстрактных рассуждений
- ❌ Формального тона

## Примеры использования

### Вопрос про стратегию
```
Вы: Как думаешь, стоит ли мне фокусироваться на AGIents
    или добить Clario до стабильности?

Claude: [Анализ через призму ваших активных проектов,
         конкретные рекомендации с учётом контекста,
         вопросы для уточнения приоритетов]
```

### Валидация идеи
```
Вы: Что скажешь про идею продуктизировать AI-код-ревью?

Claude: [Оценка через Trend+Need+Tech,
         анализ конкуренции,
         конкретные шаги для валидации,
         рекомендация стоит ли продолжать]
```

### Технический выбор
```
Вы: Поразмышляем - стоит ли переходить с VDS на AWS?

Claude: [Trade-offs анализ,
         когда имеет смысл каждый вариант,
         конкретная рекомендация для вашей стадии,
         вопрос про текущий масштаб]
```

## Thinking Frameworks

Skill использует proven frameworks под капотом:

- **First Principles** - для поиска нестандартных решений
- **Jobs To Be Done** - для валидации продуктовых идей
- **SWOT / Trade-offs** - для оценки вариантов
- **5 Whys** - для поиска root cause
- **10x vs 10%** - для оценки амбициозности

**Важно:** Фреймворки применяются неявно. Вы видите результат мышления, а не сам процесс.

## Структура ответов

Большинство ответов следуют структуре:

1. **Быстрое понимание** - подтверждение что понял вопрос
2. **Глубокий анализ** - рассмотрение с разных сторон
3. **Варианты** - 2-3 подхода с плюсами/минусами
4. **Рекомендация** - конкретное предложение
5. **Вопрос** - для продолжения диалога

## Когда НЕ использовать

Skill не подходит для:
- ❌ Простых yes/no вопросов (ответ очевиден)
- ❌ Чисто технических вопросов (нужен код, а не обсуждение)
- ❌ Когда нужна глубокая философская рефлексия (используйте thinking-partner)

## Активация

Автоматически при фразах:
- "Как думаешь..."
- "Что скажешь про..."
- "Давай поразмышляем..."
- "Нужно твоё мнение..."
- "Помоги подумать..."

Или явно через:
```
/brainstorming
```
