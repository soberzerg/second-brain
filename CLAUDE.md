## Правила именования директорий

Формат: `{число}_{Название}/` — числовой префикс + PascalCase через `_`.

| Уровень | Префикс | Шаг | Пример |
|---------|---------|-----|--------|
| 1 (корень) | сотни: `000`, `100`, `200`… | 100 | `200_Projects/` |
| 2 | десятки внутри сотни: `210`, `220`… | 10 | `210_AGIency/` |
| 3 | единицы внутри десятка: `211`, `212`… | 1 | `211_Leads/` |

- Исключение: `100_Tasks/` — внутри задачи пронумерованные от 0 до 100 (`NNN_Task`)
- Между числами оставлять зазор для будущих вставок (шаг 10 на 2-м уровне, шаг 1 на 3-м)
- Не создавать уровни глубже 3-го — вместо этого использовать плоскую структуру с файлами

## Рабочие процессы

### Начало сессии

- Проверить `000_Inbox/` на новые элементы

### Создание файлов

При создании любого `.md` файла (кроме `README.md`) **обязательно** добавлять YAML-заголовок и блок связанных документов.

#### YAML frontmatter

```yaml
---
type: <тип>                  # тип документа (обязательно), см. таблицу ниже
date: YYYY-MM-DD             # дата создания (обязательно)
source: <url>                # источник, если контент внешний (опционально)
author: <автор>              # автор источника (опционально)
project: <проект>            # связанный проект: AGIency | AGIents | Clario | MicroSaaS (опционально)
status: <статус>             # статус документа (опционально): draft | active | done | archived
tags: [тег-1, тег-2]         # теги для поиска и фильтрации (не обязательно)
---
```

#### Типы документов (`type`)

| Тип | Где используется | Пример |
|-----|-----------------|--------|
| `task` | 100_Tasks | Задача из списка 100 |
| `project-doc` | 200_Projects | Документация проекта |
| `content-idea` | 310_Content_Ideas | Идея для контента |
| `research` | 330_Research, 500_Research | Исследование |
| `lead-magnet` | 340_Lead_Magnets | Лид-магнит |

| `weekly-plan` | 410_Weekly_Plans | Недельный план |
| `weekly-synthesis` | 420_Weekly_Syntheses | Итоги недели |
| `daily-note` | 430_Daily_Notes | Ежедневная заметка |

В конце файла добавлять секцию со ссылками на связанные документы в формате Obsidian wiki-links:

```markdown

## Связанные документы

- [[Название связанного документа 1]] — краткое пояснение связи
- [[Название связанного документа 2]] — краткое пояснение связи
```
